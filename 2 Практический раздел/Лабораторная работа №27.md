# Лабораторная работа № 27  
**Реализация фрагментов для больших интерфейсов**

---

## Цель и задачи

1. Познакомить студентов с принципом разбиения больших пользовательских интерфейсов на независимые фрагменты (модули, компоненты).  
2. Научить использовать библиотеку `tkinter` (или аналогичную) для динамического создания и управления фрагментами.  
3. Развить навыки проектирования интерфейса по принципу «модульность» и «сокрытие деталей».  
4. Оценить эффективность работы интерфейса с помощью простых метрик (время загрузки, память, отклик).

---

## Теоретическая часть

| Понятие | Определение |
|---------|-------------|
| **Фрагмент** | Независимый компонент пользовательского интерфейса, содержащий собственный набор виджетов и логику. |
| **Контейнер** | Виджет, в котором размещаются фрагменты (например, `Frame` в `tkinter`). |
| **Координация** | Механизм обмена данными между фрагментами через контроллер или событийную систему. |
| **Lazy‑loading** | Отложенная инициализация фрагмента при первом запросе к нему. |
| **Событийный цикл** | Непрерывный процесс обработки пользовательских событий и обновления интерфейса. |
| **MVC** | Модель‑Вид‑Контроллер – архитектурный шаблон, разделяющий данные, представление и логику. |
| **Кеширование** | Сохранение ранее созданных фрагментов в памяти для быстрого повторного использования. |
| **Потокобезопасность** | Гарантия корректного доступа к UI‑компонентам из разных потоков. |
| **Размер окна** | Параметры `width` и `height` виджетов, влияющие на визуальное распределение фрагментов. |
| **События** | `ButtonPress`, `KeyPress`, `FocusIn`, `FocusOut` и др., используемые для взаимодействия с пользователем. |

## Ход выполнения работы

1. **Подготовка среды**  
   - Установить Python 3.10+ и `tkinter` (входит в стандартную библиотеку).  
   - Создать структуру проекта: `main.py`, `fragments/`, `controllers/`.

2. **Создание базового окна**  
   ```python
   import tkinter as tk

   root = tk.Tk()
   root.title("Большой интерфейс")
   root.geometry("800x600")
   ```

3. **Разработка шаблона фрагмента**  
   - В каталоге `fragments` создать файл `base_fragment.py`.  
   - Определить класс `BaseFragment`, наследующий `tk.Frame`.  
   ```python
   class BaseFragment(tk.Frame):
       def __init__(self, parent, controller, **kwargs):
           super().__init__(parent, **kwargs)
           self.controller = controller
           self.init_ui()

       def init_ui(self):
           pass  # реализовать в наследниках
   ```

4. **Создание конкретных фрагментов**  
   - `HeaderFragment` – шапка с навигацией.  
   - `SidebarFragment` – боковое меню.  
   - `ContentFragment` – основная область.  
   ```python
   class HeaderFragment(BaseFragment):
       def init_ui(self):
           label = tk.Label(self, text="Заголовок", font=("Arial", 16))
           label.pack(side=tk.LEFT, padx=10)
   ```

5. **Контроллер**  
   - В каталоге `controllers` создать `main_controller.py`.  
   - Класс `MainController` хранит ссылки на фрагменты и управляет их видимостью.  
   ```python
   class MainController:
       def __init__(self, root):
           self.root = root
           self.fragments = {}
           self.init_fragments()

       def init_fragments(self):
           self.fragments['header'] = HeaderFragment(self.root, self)
           self.fragments['header'].pack(fill=tk.X)

           # Аналогично для sidebar и content
   ```

6. **Lazy‑loading**  
   - В контроллере добавить метод `load_fragment(name)` – при первом вызове создаёт фрагмент, иначе возвращает уже существующий.  

7. **Подключение событий**  
   - В `SidebarFragment` добавить кнопки, при нажатии вызывающие `controller.show_content('page_name')`.  
   ```python
   btn = tk.Button(self, text="Страница 1",
                   command=lambda: self.controller.show_content('page1'))
   ```

8. **Запуск приложения**  
   ```python
   if __name__ == "__main__":
       root = tk.Tk()
       controller = MainController(root)
       root.mainloop()
   ```

9. **Анализ производительности**  
   - Сравнить время загрузки при полной и частичной инициализации фрагментов.  
   - Использовать модуль `time` для измерений.  

10. **Подготовка отчёта** (см. раздел ниже).
