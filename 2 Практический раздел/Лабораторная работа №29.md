# Лабораторная работа № 29  
**Использование Design Support Library для создания жестов**

## Цель и задачи
1. Познакомиться с библиотекой Design Support Library (DSL) и её возможностями для работы с жестами.  
2. Научиться реализовывать обработку жестов (поворот, масштабирование, свайп) в Android‑приложении.  
3. Оценить влияние реализации жестов на пользовательский интерфейс и производительность.  
4. Подготовить отчёт, включающий код, графики и выводы о проделанной работе.

## Теоретическая часть
| Понятие | Определение | Ключевое значение |
|---------|-------------|--------------------|
| Design Support Library (DSL) | Набор вспомогательных компонентов и утилит для Android, обеспечивающих совместимость UI‑элементов с разными версиями ОС. | Позволяет быстро внедрять Material Design в приложение. |
| GestureDetector | Класс, анализирующий события касания и выдающий колбэки по типу жеста. | Упрощает реализацию свайпов, двойных тапов и др. |
| ScaleGestureDetector | Специализированный детектор масштабирования, выдающий колбэки по изменению расстояния между пальцами. | Обеспечивает поддержку pinch‑zoom. |
| View.OnTouchListener | Интерфейс, позволяющий перехватывать события касания в пределах `View`. | Позволяет напрямую управлять жестами без дополнительных компонентов. |
| MotionEvent | Класс, содержащий данные о конкретном событии касания (тип, координаты, давление). | База для всех обработчиков жестов. |
| onTouchEvent() | Метод, вызываемый системой при каждом касании. | Основной вход для обработки событий. |
| GestureListener | Внутренний класс `GestureDetector.SimpleOnGestureListener`, реализующий методы `onFling()`, `onScroll()` и др. | Переопределяется для конкретной логики. |
| ScaleGestureDetector.OnScaleGestureListener | Интерфейс, определяющий методы `onScale()`, `onScaleBegin()`, `onScaleEnd()`. | Используется для обработки pinch‑zoom. |
| RecyclerView | Компонент списка с поддержкой жестов (перетаскивание, удаление). | Позволяет реализовать жесты в списках. |
| CoordinatorLayout | Контейнер, управляемый DSL, поддерживающий сложные взаимодействия (скролл, FAB). | Обеспечивает плавные анимации при жестах. |

## Ход выполнения работы
1. **Создание проекта**  
   - Открыть Android Studio, создать новый проект с Activity `Empty Activity`.  
   - В `build.gradle` добавить зависимость:  
     ```gradle
     implementation 'com.google.android.material:material:1.9.0'
     ```
2. **Добавление UI‑элементов**  
   - В `activity_main.xml` разместить `CoordinatorLayout` → `AppBarLayout` → `Toolbar` → `FrameLayout` для контента.  
   - Внутри `FrameLayout` разместить `ImageView` (для демонстрации pinch‑zoom).  
3. **Настройка GestureDetector**  
   - В `MainActivity.kt` создать объект `GestureDetector`:  
     ```kotlin
     private lateinit var gestureDetector: GestureDetector
     
     override fun onCreate(savedInstanceState: Bundle?) {
         super.onCreate(savedInstanceState)
         setContentView(R.layout.activity_main)
         
         gestureDetector = GestureDetector(this, object : GestureDetector.SimpleOnGestureListener() {
             override fun onFling(e1: MotionEvent, e2: MotionEvent, velocityX: Float, velocityY: Float): Boolean {
                 // логика свайпа
                 return true
             }
         })
     }
     ```
4. **Настройка ScaleGestureDetector**  
   - Добавить объект `ScaleGestureDetector`:  
     ```kotlin
     private lateinit var scaleDetector: ScaleGestureDetector
     
     init {
         scaleDetector = ScaleGestureDetector(this, object : ScaleGestureDetector.OnScaleGestureListener {
             override fun onScale(detector: ScaleGestureDetector): Boolean {
                 val scale = detector.scaleFactor
                 imageView.scaleX *= scale
                 imageView.scaleY *= scale
                 return true
             }
             override fun onScaleBegin(detector: ScaleGestureDetector): Boolean = true
             override fun onScaleEnd(detector: ScaleGestureDetector) {}
         })
     }
     ```
5. **Перехват событий касания**  
   - Установить `OnTouchListener` для `ImageView`:  
     ```kotlin
     imageView.setOnTouchListener { v, event ->
         gestureDetector.onTouchEvent(event)
         scaleDetector.onTouchEvent(event)
         true
     }
     ```
6. **Тестирование**  
   - Запустить приложение на эмуляторе/устройстве.  
   - Проверить работу свайпа (добавить вывод в Logcat).  
   - Проверить масштабирование изображения pinch‑zoom.  
7. **Анализ результатов**  
   - Оценить плавность анимаций, отклик на жесты.  
   - Сравнить с базовой реализацией без DSL (если возможно).
