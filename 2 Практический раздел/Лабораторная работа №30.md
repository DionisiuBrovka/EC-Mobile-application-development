# Лабораторная работа № 30  
Создание списков на основе RecyclerView  

## Цель и задачи  
1. Ознакомиться с принципами работы списков в пользовательском интерфейсе.  
2. Научиться реализовать адаптер, ViewHolder и RecyclerView для отображения больших наборов данных.  
3. Разобрать способы оптимизации работы списка: переиспользование ячеек, обновление данных.  
4. Разработать простое приложение, демонстрирующее динамическое добавление и удаление элементов списка.  

## Теоретическая часть  
| Понятие | Определение | Ключевой момент |
|---------|-------------|-----------------|
| RecyclerView | Компонент UI, позволяющий эффективно отображать большие наборы данных, повторно используя ячейки. | Переиспользование ViewHolder снижает потребление памяти. |
| Adapter | Класс, связывающий данные с представлением элементов списка. | `onCreateViewHolder`, `onBindViewHolder`, `getItemCount`. |
| ViewHolder | Паттерн, сохраняющий ссылки на представления элемента списка. | Уменьшает количество вызовов `findViewById`. |
| LayoutManager | Управляет размещением элементов в RecyclerView (Linear, Grid, Staggered). | Позволяет менять ориентацию списка. |
| DiffUtil | Утилита, рассчитывающая минимальные изменения между двумя списками. | Оптимизирует обновление UI. |
| Payload | Данные, передаваемые в `onBindViewHolder` при частичном обновлении. | Позволяет обновлять только часть элемента. |
| ItemDecoration | Класс, добавляющий отступы, разделители, маркеры к элементам списка. | Улучшает визуальное восприятие. |
| AdapterDataObserver | Слушатель изменений данных адаптера. | Позволяет реагировать на обновления. |
| ViewBinding / DataBinding | Технологии связывания XML с кодом. | Уменьшают вероятность ошибок при доступе к UI. |
| LiveData / Flow | Реактивные потоки данных, часто используемые с RecyclerView. | Автоматически обновляют UI при изменении данных. |

## Ход выполнения работы  

1. **Создание проекта**  
   - Откройте IDE (Android Studio, PyCharm с плагином для Android, или любой редактор).  
   - Создайте новый проект с шаблоном “Empty Activity”.  
   ```bash
   $ android create project -a empty
   ```
2. **Добавление зависимостей**  
   В `build.gradle (Module)` добавьте библиотеки:  
   ```gradle
   implementation 'androidx.recyclerview:recyclerview:1.2.1'
   implementation 'androidx.lifecycle:lifecycle-livedata-ktx:2.5.1'
   implementation 'androidx.activity:activity-ktx:1.7.2'
   ```
3. **Разметка XML**  
   В `activity_main.xml` разместите `RecyclerView`.  
   ```xml
   <androidx.recyclerview.widget.RecyclerView
       android:id="@+id/recyclerView"
       android:layout_width="match_parent"
       android:layout_height="match_parent"
       android:layout_margin="16dp" />
   ```
4. **Модель данных**  
   Создайте класс `Item`.  
   ```kotlin
   data class Item(val id: Int, val title: String, val description: String)
   ```
5. **Создание ViewHolder**  
   ```kotlin
   class ItemViewHolder(private val binding: ItemLayoutBinding) :
       RecyclerView.ViewHolder(binding.root) {
       fun bind(item: Item) {
           binding.titleText.text = item.title
           binding.descText.text = item.description
       }
   }
   ```
6. **Создание Adapter**  
   ```kotlin
   class ItemAdapter : ListAdapter<Item, ItemViewHolder>(ItemDiffCallback()) {
       override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): ItemViewHolder {
           val binding = ItemLayoutBinding.inflate(LayoutInflater.from(parent.context), parent, false)
           return ItemViewHolder(binding)
       }
       override fun onBindViewHolder(holder: ItemViewHolder, position: Int) {
           holder.bind(getItem(position))
       }
   }

   class ItemDiffCallback : DiffUtil.ItemCallback<Item>() {
       override fun areItemsTheSame(oldItem: Item, newItem: Item) = oldItem.id == newItem.id
       override fun areContentsTheSame(oldItem: Item, newItem: Item) = oldItem == newItem
   }
   ```
7. **Подключение Adapter к RecyclerView**  
   ```kotlin
   class MainActivity : AppCompatActivity() {
       private lateinit var binding: ActivityMainBinding
       private val adapter = ItemAdapter()
       override fun onCreate(savedInstanceState: Bundle?) {
           super.onCreate(savedInstanceState)
           binding = ActivityMainBinding.inflate(layoutInflater)
           setContentView(binding.root)
           binding.recyclerView.layoutManager = LinearLayoutManager(this)
           binding.recyclerView.adapter = adapter
           loadData()
       }
       private fun loadData() {
           val items = List(100) { Item(it, "Title $it", "Description $it") }
           adapter.submitList(items)
       }
   }
   ```
8. **Динамическое изменение списка**  
   Добавьте кнопку “Добавить” и “Удалить” в `activity_main.xml`.  
   ```kotlin
   binding.addButton.setOnClickListener {
       val newList = adapter.currentList.toMutableList()
       val newId = (adapter.currentList.maxOfOrNull { it.id } ?: 0) + 1
       newList.add(Item(newId, "New Title", "New Description"))
       adapter.submitList(newList)
   }
   ```
9. **Тестирование**  
   - Запустите приложение.  
   - Проверьте плавность прокрутки.  
   - Добавьте и удалите элементы, убедитесь, что список обновляется корректно.  
10. **Анализ результатов**  
    - Оцените использование памяти до и после добавления 1000 элементов.  
    - Сравните время загрузки списка с и без `DiffUtil`.

7. Как добавить разделитель между элементами списка?  
8. Какие данные можно передавать в `onBindViewHolder` через `payloads`?
