# Лабораторная работа № 14  
Обработка исключительных ситуаций  

## Цель и задачи  
1. Ознакомиться с механизмом обработки исключений в Python.  
2. Научиться использовать конструкции `try…except`, `else`, `finally` и создавать собственные исключения.  
3. Разработать программы, в которых корректно обрабатываются возможные ошибки ввода/вывода, деления и некорректных данных.  

## Теоретическая часть  
1. **Исключение** – событие, возникающее при выполнении программы, которое нарушает её обычный ход.  
2. **Блок `try`** – участок кода, в котором может возникнуть исключение.  
3. **Блок `except`** – обработчик конкретного типа исключения.  
4. **Блок `else`** – исполняется, если в блоке `try` не возникло исключения.  
5. **Блок `finally`** – исполняется всегда, независимо от того, было ли исключение.  
6. **Класс `BaseException`** – корневой класс всех исключений в Python.  
7. **Класс `Exception`** – базовый класс для пользовательских исключений.  
8. **Переопределение `__init__`** – позволяет задать собственные атрибуты и сообщение исключения.  
9. **Функция `raise`** – генерирует исключение.  
10. **Иерархия исключений** – `ZeroDivisionError`, `ValueError`, `TypeError`, `IOError` и др.  

## Ход выполнения работы  

1. **Создание простого обработчика**  
   - Описываем, что произойдет при делении на ноль.  
   ```python
   try:
       a = 10
       b = 0
       print(a / b)
   except ZeroDivisionError as e:
       print(f"Ошибка: {e}")
   ```

2. **Обработка нескольких исключений**  
   - Обрабатываем `ValueError` и `TypeError` в одном блоке.  
   ```python
   try:
       x = int(input("Введите целое число: "))
       print(100 / x)
   except (ValueError, ZeroDivisionError) as e:
       print(f"Неверные данные: {e}")
   ```

3. **Использование `else` и `finally`**  
   - `else` выводит результат, если ошибок нет.  
   - `finally` закрывает файл независимо от ошибок.  
   ```python
   try:
       f = open("data.txt", "r")
   except FileNotFoundError:
       print("Файл не найден")
   else:
       content = f.read()
       print(content)
   finally:
       f.close()
   ```

4. **Создание собственного исключения**  
   - Определяем класс `NegativeNumberError`.  
   ```python
   class NegativeNumberError(Exception):
       def __init__(self, value):
           super().__init__(f"Значение {value} отрицательно")
   ```

5. **Вызов пользовательского исключения**  
   ```python
   def square_root(x):
       if x < 0:
           raise NegativeNumberError(x)
       return x ** 0.5

   try:
       print(square_root(-9))
   except NegativeNumberError as e:
       print(e)
   ```

6. **Экспериментальная часть**  
   - Создайте программу, которая запрашивает у пользователя путь к файлу, читает его и выводит первую строку.  
   - Обработайте исключения: `FileNotFoundError`, `PermissionError`, `IOError`.  
   - Проведите несколько экспериментов:  
     1. Путь к существующему файлу.  
     2. Путь к несуществующему файлу.  
     3. Путь к файлу без прав чтения.  
   - Запишите в отчёт выводы по каждому случаю.

6. Как гарантировать закрытие файла, даже если возникло исключение?
