# Лабораторная работа № 38  
Создание приемников широковещательных намерений  

## Цель и задачи  
1. Ознакомиться с принципами работы широковещательных сообщений (broadcast intents) в Android.  
2. Научиться регистрировать и обрабатывать BroadcastReceiver в коде и через манифест.  
3. Разработать простое приложение, которое реагирует на системные события (изменение времени, подключение к Wi‑Fi, выключение устройства).  
4. Оценить эффективность использования динамических и статических приемников в различных сценариях.

## Теоретическая часть  
| Понятие | Описание |
|---------|----------|
| **Intent** | Абстракция сообщения, передающая действие и данные между компонентами. |
| **Broadcast Intent** | Специфический Intent, который может быть отправлен нескольким приёмникам. |
| **BroadcastReceiver** | Класс, реализующий `onReceive(Context, Intent)`, обрабатывающий широковещательные сообщения. |
| **Stateless Receiver** | Приёмник, объявленный в манифесте, активируется системой независимо от состояния приложения. |
| **Stateful Receiver** | Приёмник, зарегистрированный в рантайме через `Context.registerReceiver()`. |
| **Filter** | `IntentFilter` указывает, какие намерения принимает BroadcastReceiver. |
| **System Broadcast** | Предопределённые Intent, отправляемые ОС (e.g., `ACTION_BATTERY_CHANGED`). |
| **Custom Broadcast** | Пользовательские Intent, отправляемые приложением (`LocalBroadcastManager`). |
| **Permission** | Разрешение, необходимое для получения определённых системных broadcast (e.g., `android.permission.ACCESS_WIFI_STATE`). |
| **Receiver Priority** | Приоритет приёмника, определяющий порядок вызова при совпадении фильтра. |

## Ход выполнения работы  

1. **Подготовка проекта**  
   - Создайте новый Android‑Studio проект (`Empty Activity`).  
   - В `build.gradle` добавьте зависимость `implementation 'androidx.localbroadcastmanager:localbroadcastmanager:1.0.0'`.  
   ```gradle
   dependencies {
       implementation 'androidx.localbroadcastmanager:localbroadcastmanager:1.0.0'
   }
   ```

2. **Определение статического приёмника**  
   - В `AndroidManifest.xml` объявите `BroadcastReceiver` с `intent-filter`.  
   ```xml
   <receiver android:name=".BatteryReceiver" android:exported="false">
       <intent-filter>
           <action android:name="android.intent.action.BATTERY_CHANGED"/>
       </intent-filter>
   </receiver>
   ```  
   - Создайте класс `BatteryReceiver.kt` и реализуйте `onReceive`.  
   ```kotlin
   class BatteryReceiver : BroadcastReceiver() {
       override fun onReceive(context: Context, intent: Intent) {
           val level = intent.getIntExtra(BatteryManager.EXTRA_LEVEL, -1)
           Log.d("BatteryReceiver", "Battery level: $level%")
       }
   }
   ```

3. **Регистрация динамического приёмника**  
   - В `MainActivity.kt` объявите `BroadcastReceiver` и зарегистрируйте его в `onResume`, отрегиструйте в `onPause`.  
   ```kotlin
   class MainActivity : AppCompatActivity() {
       private val wifiReceiver = object : BroadcastReceiver() {
           override fun onReceive(context: Context, intent: Intent) {
               val state = intent.getIntExtra(WifiManager.EXTRA_WIFI_STATE, -1)
               Log.d("WiFiReceiver", "WiFi state: $state")
           }
       }

       override fun onResume() {
           super.onResume()
           val filter = IntentFilter(WifiManager.WIFI_STATE_CHANGED_ACTION)
           registerReceiver(wifiReceiver, filter)
       }

       override fun onPause() {
           super.onPause()
           unregisterReceiver(wifiReceiver)
       }
   }
   ```

4. **Отправка пользовательского широковещательного сообщения**  
   - Создайте кнопку в `activity_main.xml`.  
   - В `onClick` отправьте локальный broadcast.  
   ```kotlin
   val btnSend = findViewById<Button>(R.id.btnSend)
   btnSend.setOnClickListener {
       val intent = Intent("com.example.CUSTOM_BROADCAST")
       intent.putExtra("message", "Hello, Broadcast!")
       LocalBroadcastManager.getInstance(this).sendBroadcast(intent)
   }
   ```  
   - Зарегистрируйте `LocalBroadcastManager`‑приёмник аналогично динамическому.  

5. **Проверка работы**  
   - Запустите приложение на устройстве/эмуляторе.  
   - Проверьте вывод в Logcat для каждого приёмника.  
   - Подключите/отключите Wi‑Fi, измените заряд батареи, убедитесь, что статический и динамический приёмники реагируют.  

6. **Анализ результатов**  
   - Сравните время реакции и потребление ресурсов между статическим и динамическим приёмниками.  
   - Оцените необходимость использования `exported="false"` и `permission`.  

## Контрольные вопросы / задания для самопроверки  

1. Что такое `IntentFilter` и какие элементы он может содержать?  
2. В чем разница между статическим и динамическим `BroadcastReceiver`?  
3. Какую роль играет атрибут `exported` в манифесте приёмника?  
4. Как отправить пользовательский broadcast через `LocalBroadcastManager`?  
5. Какие системные события чаще всего используют в практических задачах?  
6. Почему динамический приёмник рекомендуется регистрировать в `onResume()` и удалять в `onPause()`?  
7. Какие разрешения нужны для получения `ACTION_BATTERY_CHANGED`?  
8. Как задать приоритет приёмника и как это влияет на порядок вызова?  
9. Что произойдёт, если два приёмника регистрируются с одинаковым фильтром и одинаковым приоритетом?  
10. Напишите код, который отреагирует только на событие включения Wi‑Fi.
