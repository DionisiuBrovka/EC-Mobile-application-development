# Лабораторная работа № 8  
Разработка программ, реализующих механизм наследования  

## Цель и задачи  
1. Изучить принцип работы наследования в объектно‑ориентированном программировании.  
2. Научиться создавать и использовать базовый и производный классы.  
3. Понять, как переопределять методы и использовать `super()`.  
4. Разработать программу, демонстрирующую преимущества наследования.  

## Теоретическая часть  
| Понятие | Определение | Пример |
|---------|-------------|--------|
| Класс | Шаблон (тип) объектов, описывающий атрибуты и поведение. | `class Animal:` |
| Объект | Конкретный экземпляр класса. | `cat = Cat()` |
| Наследование | Механизм, позволяющий создавать новый класс (производный) на основе существующего (базового). | `class Cat(Animal):` |
| Переопределение метода | Замена реализации метода в производном классе. | `def speak(self):` |
| `super()` | Функция, возвращающая ссылку на базовый класс для вызова его методов. | `super().speak()` |
| Полиморфизм | Возможность вызывать один и тот же метод у объектов разных типов. | `animals = [Cat(), Dog()]` |
| Инкапсуляция | Сокрытие внутренней реализации класса. | `self.__age` |
| Абстрактный класс | Класс, который не может быть инстанцирован и содержит абстрактные методы. | `from abc import ABC, abstractmethod` |
| Множественное наследование | Наследование от более чем одного базового класса. | `class FlyingDog(Dog, Flyer):` |
| Инициализация (`__init__`) | Специальный метод, вызываемый при создании объекта. | `def __init__(self, name):` |

## Ход выполнения работы  

1. **Создание базового класса**  
   - Определяем класс `Shape` с атрибутом `color` и методом `area()`, который возвращает `NotImplementedError`.  
   ```python
   class Shape:
       def __init__(self, color):
           self.color = color

       def area(self):
           raise NotImplementedError("Метод area должен быть переопределен")
   ```

2. **Создание производных классов**  
   - Класс `Circle` наследует от `Shape`, добавляет атрибут `radius` и реализует `area()`.  
   - Класс `Rectangle` наследует от `Shape`, добавляет `width`, `height` и реализует `area()`.  
   ```python
   class Circle(Shape):
       def __init__(self, color, radius):
           super().__init__(color)
           self.radius = radius

       def area(self):
           return 3.14159 * self.radius ** 2

   class Rectangle(Shape):
       def __init__(self, color, width, height):
           super().__init__(color)
           self.width = width
           self.height = height

       def area(self):
           return self.width * self.height
   ```

3. **Переопределение метода `__str__`**  
   - В каждом производном классе реализуем `__str__`, чтобы выводить информацию о фигуре.  
   ```python
   def __str__(self):
       return f"{self.__class__.__name__} (color={self.color})"
   ```

4. **Создание списка объектов и полиморфный вызов**  
   - Создаём список `shapes = [Circle('red', 5), Rectangle('blue', 4, 6)]`.  
   - Для каждого объекта выводим его строковое представление и площадь.  
   ```python
   for shape in shapes:
       print(shape)
       print("Area:", shape.area())
   ```

5. **Экспериментальная часть**  
   - Изменяем цвет базового объекта, наблюдаем влияние на производные.  
   - Добавляем новый класс `Triangle`, демонстрируя расширение и переопределение.  
   ```python
   class Triangle(Shape):
       def __init__(self, color, base, height):
           super().__init__(color)
           self.base = base
           self.height = height

       def area(self):
           return 0.5 * self.base * self.height
   ```

6. **Анализ результатов**  
   - Сравниваем площади, выводим выводы о переопределении и использовании `super()`.
