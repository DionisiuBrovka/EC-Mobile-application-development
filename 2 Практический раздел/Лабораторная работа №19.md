# Лабораторная работа № 19 Создание приложения для Android

## Цель и задачи

1. Познакомиться с принципами разработки мобильных приложений на платформе Android.  
2. Научиться использовать Python‑библиотеку Kivy для создания кросс‑платформенного GUI.  
3. Реализовать простое приложение «Счётчик» и собрать его в APK‑файл.  
4. Проанализировать полученный APK, понять структуру проекта и основные моменты сборки.

## Теоретическая часть

| Понятие | Описание |
|---------|----------|
| **Android** | Мобильная операционная система, основанная на ядре Linux. |
| **APK** | Android Package — архив, содержащий все ресурсы и исполняемый код. |
| **Kivy** | Кросс‑платформенный фреймворк для разработки GUI на Python. |
| **Widget** | Базовый элемент пользовательского интерфейса в Kivy. |
| **KV‑файл** | Файл декларации интерфейса Kivy, использующий собственный язык разметки. |
| **Buildozer** | Инструмент автоматизации сборки Kivy‑приложений под Android (и другие платформы). |
| **Python‑to‑JavaBridge** | Технология, позволяющая интегрировать Python‑код в Android‑приложение через Jython/SL4A. |
| **Gradle** | Система сборки, используемая Android Studio и Buildozer. |
| **Proguard** | Инструмент для минификации и обфускации кода, применяемый в процессе сборки. |
| **AndroidManifest.xml** | Конфигурационный файл, определяющий разрешения, компоненты и метаданные приложения. |

## Ход выполнения работы

1. **Установка окружения**  
   - Установить Python 3.10+ и `pip`.  
   - Установить `kivy` и `buildozer`:
     ```bash
     pip install kivy buildozer
     ```
   - Настроить Android SDK и NDK (путь к `ANDROID_HOME`).  

2. **Создание проекта**  
   - Создать каталог `kivy_counter`.  
   - Внутри выполнить:
     ```bash
     buildozer init
     ```
   - В `buildozer.spec` отредактировать:
     - `package.name = CounterApp`  
     - `package.domain = org.example`  
     - `source.include_exts = py,png,jpg,kv`  
     - `requirements = python3,kivy`  

3. **Разработка пользовательского интерфейса**  
   - Создать файл `main.py`:
     ```python
     from kivy.app import App
     from kivy.uix.boxlayout import BoxLayout
     from kivy.properties import NumericProperty

     class Counter(BoxLayout):
         count = NumericProperty(0)

         def increment(self):
             self.count += 1

     class CounterApp(App):
         def build(self):
             return Counter()
     ```
   - Создать файл `counter.kv`:
     ```kv
     <Counter>:
         orientation: 'vertical'
         Label:
             text: str(root.count)
             font_size: 48
         Button:
             text: 'Нажать'
             on_press: root.increment()
     ```
   - Запустить приложение в эмуляторе:
     ```bash
     python main.py
     ```

4. **Тестирование на Android‑эмуляторе**  
   - Запустить эмулятор (Android Studio > AVD Manager).  
   - Скомпилировать и запустить:
     ```bash
     buildozer -v android debug deploy run
     ```
   - Проверить работоспособность кнопки и отображение счётчика.

5. **Сборка релиз‑версии**  
   - Настроить подпись APK в `buildozer.spec` (`android.release.key`).  
   - Сгенерировать APK:
     ```bash
     buildozer android release
     ```
   - Положить APK в каталог `bin/`.

6. **Анализ APK**  
   - Распаковать APK (`apktool` или `7z`).  
   - Изучить структуру: `AndroidManifest.xml`, `classes.dex`, `assets`, `lib`.  
   - Проверить наличие ключевых файлов: `META-INF`, `lib/armeabi-v7a/libkivy.so`.

5. Как проверить, что приложение имеет доступ к интернету после сборки?

## Отчёт по работе

Отчёт должен содержать:

1. **Титульный лист** – название лабораторной, дата, имя студента.  
2. **Введение** – цель работы, актуальность темы.  
3. **Теоретическая часть** – краткое изложение ключевых понятий.  
4. **Ход выполнения** – пошаговое описание, включая используемые команды, проблемы и их решения.  
5. **Результаты** – скриншоты приложения на эмуляторе, список проверенных функций, анализ APK.  
6. **Выводы** – что было изучено, какие навыки получены, возможные улучшения.  
7. **Приложения** – исходный код (`main.py`, `counter.kv`), `buildozer.spec`, сгенерированный APK.  

## Критерии оценки

1. Полнота и корректность выполнения всех шагов.  
2. Качество кода: читаемость, использование свойств Kivy, отсутствие ошибок.  
3. Точность отчёта: наличие всех обязательных разделов и достаточная детализация.  
4. Способность проанализировать структуру APK и объяснить ключевые компоненты.  
5. Самостоятельность решения возникающих проблем при сборке и отладке.
