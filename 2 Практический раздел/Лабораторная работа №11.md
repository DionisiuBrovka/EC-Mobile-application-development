# Лабораторная работа №11  
Разработка программ с использованием пакетов

## Цель и задачи

1. Ознакомиться с концепцией пакетирования в Python.  
2. Научиться создавать собственные пакеты и подключать внешние.  
3. Разработать небольшую программу, использующую несколько пакетов, и проверить её работу.  
4. Научиться публиковать пакет в локальном репозитории (pypi) и устанавливать его через `pip`.

## Теоретическая часть

| Понятие | Описание |
|---------|----------|
| **Модуль** | Файл с расширением `.py`, содержащий функции, классы, переменные. |
| **Пакет** | Набор модулей, сгруппированных в каталог с файлом `__init__.py`. |
| **`__init__.py`** | Файл, сигнализирующий Python, что каталог является пакетом; может инициализировать пакет. |
| **Импорт** | Операция `import package.module` или `from package import module`. |
| **`setup.py`** | Скрипт конфигурации пакета (параметры `name`, `version`, `packages`, `install_requires`). |
| **`pip`** | Инструмент для установки пакетов из репозитория PyPI или локального источника. |
| **`requirements.txt`** | Файл, содержащий список зависимостей с указанием версий. |
| **`__all__`** | Список публичных объектов модуля/пакета. |
| **PEP 517/518** | Стандарты для сборки и публикации пакетов. |
| **`setuptools`** | Библиотека, упрощающая создание пакетов. |

## Ход выполнения работы

1. **Создание структуры проекта**  
   ```bash
   lab11/
   ├─ mypkg/
   │  ├─ __init__.py
   │  ├─ utils.py
   │  └─ calc.py
   ├─ tests/
   │  └─ test_mypkg.py
   ├─ setup.py
   └─ README.md
   ```
   *Пояснение:* `mypkg` – основной пакет, `tests` – модульные тесты.

2. **Написание модуля `utils.py`**  
   ```python
   # mypkg/utils.py
   def greet(name: str) -> str:
       """Возвращает приветствие."""
       return f"Hello, {name}!"
   ```

3. **Создание модуля `calc.py`**  
   ```python
   # mypkg/calc.py
   from .utils import greet

   def add(a: float, b: float) -> float:
       return a + b

   def main():
       print(greet("Student"))
       print(f"3 + 5 = {add(3, 5)}")
   ```

4. **Инициализация пакета (`__init__.py`)**  
   ```python
   # mypkg/__init__.py
   from .calc import add
   from .utils import greet

   __all__ = ["add", "greet"]
   ```

5. **Подготовка файла `setup.py`**  
   ```python
   # setup.py
   from setuptools import setup, find_packages

   setup(
       name="mypkg",
       version="0.1.0",
       packages=find_packages(),
       install_requires=[],
       python_requires=">=3.8",
       author="Your Name",
       description="Пример пакета для лабораторной работы",
   )
   ```

6. **Запуск программы**  
   ```bash
   python -m mypkg
   ```
   *Ожидаемый вывод:*
   ```
   Hello, Student!
   3 + 5 = 8
   ```

7. **Тестирование**  
   ```python
   # tests/test_mypkg.py
   import unittest
   from mypkg import add, greet

   class TestMyPkg(unittest.TestCase):
       def test_greet(self):
           self.assertEqual(greet("A"), "Hello, A!")

       def test_add(self):
           self.assertEqual(add(2, 3), 5)

   if __name__ == "__main__":
       unittest.main()
   ```
   Запуск: `python -m unittest discover tests`

8. **Публикация пакета в локальный репозиторий**  
   ```bash
   pip install --upgrade twine
   python -m build
   twine upload dist/* --repository local
   ```

9. **Установка пакета из локального репозитория**  
   ```bash
   pip install --index-url http://localhost:8080/simple/ mypkg
   ```
