# Лабораторная работа № 23 Реализация макетов с ограничениями

## Цель и задачи
1. Ознакомиться с принципами проектирования макетов (моделей) в Python.  
2. Научиться реализовывать ограничения (validation) на атрибутах классов.  
3. Освоить работу с декораторами и метаклассами для автоматизации валидации.  
4. Разработать и протестировать собственный макет, удовлетворяющий заданным ограничениям.

## Теоретическая часть
- **Класс** – определение типа данных, объединяющее атрибуты и методы.  
- **Атрибут** – переменная, принадлежащая экземпляру класса.  
- **Инициализатор** (`__init__`) – метод, выполняющийся при создании объекта.  
- **Декоратор** – функция, принимающая функцию или класс и возвращающая изменённую версию.  
- **Метакласс** – класс, создающий другие классы; реализует логику создания объектов.  
- **Валидация** – проверка корректности значений атрибутов.  
- **Константа** – неизменяемое значение, обычно объявляемое в верхнем регистре.  
- **Типизация** (`typing`) – указание типов переменных и аргументов функций.  
- **Исключение** (`Exception`) – объект, сигнализирующий об ошибке выполнения.  
- **Контекстный менеджер** (`with`) – обеспечивает корректное открытие/закрытие ресурсов.  
- **Модуль `dataclasses`** – упрощает создание классов с автоматически генерируемыми методами.

## Ход выполнения работы

1. **Подготовка среды**  
   - Установить Python 3.9+ и IDE (PyCharm, VS Code).  
   - Создать виртуальное окружение:  
     ```bash
     python -m venv venv
     source venv/bin/activate  # Linux/macOS
     venv\Scripts\activate     # Windows
     ```

2. **Определение базового макета**  
   - Создать файл `model.py`.  
   - Реализовать класс `BaseModel` с атрибутом `_data` и методом `save()`.  
     ```python
     from dataclasses import dataclass, field
     from typing import Dict, Any

     @dataclass
     class BaseModel:
         _data: Dict[str, Any] = field(default_factory=dict)

         def save(self) -> None:
             """Сохраняет данные в файл JSON."""
             import json
             with open(f"{self.__class__.__name__}.json", "w") as f:
                 json.dump(self._data, f, indent=4)
     ```

3. **Создание механизма валидации**  
   - Написать декоратор `validate` для проверки значений атрибутов.  
     ```python
     def validate(func):
         def wrapper(self, *args, **kwargs):
             for key, value in kwargs.items():
                 if key not in self._fields:
                     raise ValueError(f"Неизвестное поле: {key}")
                 validator = self._validators.get(key)
                 if validator and not validator(value):
                     raise ValueError(f"Недопустимое значение для {key}: {value}")
             return func(self, *args, **kwargs)
         return wrapper
     ```

4. **Определение конкретного макета**  
   - Создать класс `Person` наследующий `BaseModel`.  
   - Задать поля: `name` (str, не пустой), `age` (int, 0–120).  
     ```python
     class Person(BaseModel):
         _fields = {"name", "age"}
         _validators = {
             "name": lambda v: isinstance(v, str) and bool(v.strip()),
             "age": lambda v: isinstance(v, int) and 0 <= v <= 120
         }

         @validate
         def __init__(self, **kwargs):
             super().__init__()
             self._data.update(kwargs)
     ```

5. **Тестирование**  
   - В файле `test_person.py` написать несколько тестов с помощью `unittest`.  
     ```python
     import unittest
     from model import Person

     class TestPerson(unittest.TestCase):
         def test_valid(self):
             p = Person(name="Иван", age=30)
             self.assertEqual(p._data["name"], "Иван")

         def test_invalid_age(self):
             with self.assertRaises(ValueError):
                 Person(name="Анна", age=150)

     if __name__ == "__main__":
         unittest.main()
     ```

6. **Экспериментальная часть**

   - Сохранить корректный объект с помощью `save()`.  
   - Проверить наличие файла `Person.json` и его содержимое.

7. **Анализ результатов**
