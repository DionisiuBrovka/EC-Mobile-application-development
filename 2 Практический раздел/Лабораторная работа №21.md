# Лабораторная работа № 21  
Обработка жизненного цикла активности

## Цель и задачи

1. Научиться использовать менеджер контекста `with` и декоратор `@contextmanager` для управления ресурсами.  
2. Понять принципы работы цикла событий в GUI‑приложениях (обработчики событий, главная петля).  
3. Разработать простое приложение с использованием библиотеки `tkinter`, в котором реализован контроль жизненного цикла окна.  
4. Оценить влияние корректного закрытия ресурсов на стабильность программы.

## Теоретическая часть

| Понятие | Определение |
|---------|-------------|
| **Ресурс** | Любой объект, требующий явного освобождения (файлы, сокеты, окна). |
| **Менеджер контекста** | Класс, реализующий методы `__enter__` и `__exit__`. Позволяет гарантировать освобождение ресурса. |
| **Декоратор `@contextmanager`** | Функция‑генератор, преобразуемая в менеджер контекста. |
| **Главная петля (`mainloop`)** | Бесконечный цикл, обрабатывающий события GUI‑приложения. |
| **Обработчик событий** | Функция, привязанная к событию (нажатие кнопки, закрытие окна). |
| **`tkinter.Tk`** | Класс, создающий главное окно приложения. |
| **`protocol("WM_DELETE_WINDOW", handler)`** | Привязка пользовательского обработчика к событию закрытия окна. |
| **`destroy()`** | Метод, закрывающий окно и завершающий главную петлю. |
| **`finally`** | Блок, гарантируемый исполняется независимо от исключений. |
| **`__del__`** | Деструктор объекта, вызывается при сборке мусора. |

## Ход выполнения работы

1. **Создание проекта**  
   - Создайте новый файл `activity_lifecycle.py`.  
   - Импортируйте необходимые модули:  
     ```python
     import tkinter as tk
     from contextlib import contextmanager
     ```

2. **Определение менеджера контекста для окна**  
   - Реализуйте `@contextmanager`‑функцию `managed_window()`, которая создаёт объект `tk.Tk`, регистрирует обработчик закрытия и гарантирует вызов `destroy()`.  
     ```python
     @contextmanager
     def managed_window():
         root = tk.Tk()
         try:
             yield root
         finally:
             root.destroy()
     ```

3. **Функция‑обработчик закрытия**  
   - Создайте функцию `on_close()` с логикой, которая будет выполнена при попытке закрыть окно.  
     ```python
     def on_close():
         print("Окно закрывается")
     ```

4. **Основной блок программы**  
   - Используйте менеджер контекста, создайте кнопку и привяжите обработчик событий.  
     ```python
     if __name__ == "__main__":
         with managed_window() as root:
             root.protocol("WM_DELETE_WINDOW", on_close)
             btn = tk.Button(root, text="Нажми меня")
             btn.pack(padx=20, pady=20)
             root.mainloop()
     ```

5. **Тестирование**  
   - Запустите программу.  
   - Нажмите кнопку – должно появиться сообщение в консоли.  
   - Попробуйте закрыть окно – увидите вывод `Окно закрывается` и завершение программы.  
   - Убедитесь, что после закрытия окна не остаётся активных процессов.

## Контрольные вопросы / задания для самопроверки

1. Что делает метод `destroy()` в `tkinter`?

3. В чем отличие менеджера контекста, созданного через `@contextmanager`, от обычного класса‑менеджера?  
4. Какой протокол используется для привязки обработчика закрытия окна?  
5. Почему важно вызывать `mainloop()` внутри блока `with`?  
6. Что произойдёт, если убрать `finally` в функции‑менеджере?

10. Что такое «жизненный цикл активности» в контексте данной лабораторной работы?
