# Лабораторная работа № 9  
Разработка программ с использованием наследования и переопределения методов  

## Цель и задачи  
1. Познакомиться с принципами объектно‑ориентированного программирования: наследование, полиморфизм и переопределение методов.  
2. Научиться создавать иерархию классов, реализовывать наследование и переопределять методы.  
3. Практиковаться в написании тестов и проверке корректности работы наследуемых классов.  
4. Развить навыки документирования кода и написания отчёта по лабораторной работе.  

## Теоретическая часть  
1. **Класс** – шаблон, определяющий атрибуты и поведение объектов.  
2. **Объект** – конкретный экземпляр класса.  
3. **Наследование** – механизм, позволяющий одному классу (наследнику) получить атрибуты и методы другого (родителя).  
4. **Переопределение метода** – переопределение поведения метода в наследнике с тем же именем и сигнатурой.  
5. **super()** – функция, возвращающая ссылку на родительский класс для вызова его методов.  
6. **Абстрактный класс** – класс, не предназначенный для непосредственного создания объектов, но определяющий интерфейс.  
7. **Интерфейс** – набор абстрактных методов, которые должен реализовать класс.  
8. **Полиморфизм** – способность объектов разных классов отвечать на одинаковый вызов метода по-своему.  
9. **Множественное наследование** – наследование от более чем одного родительского класса (в Python допустимо).  
10. **Миксин** – небольшая вспомогательная часть кода, которую можно добавить к разным классам для расширения функционала.  

## Ход выполнения работы  

1. **Создание базового класса**  
   - Определяем класс `Shape` с абстрактным методом `area`.  
   ```python
   from abc import ABC, abstractmethod

   class Shape(ABC):
       @abstractmethod
       def area(self) -> float:
           """Вычисление площади фигуры."""
           pass
   ```

2. **Наследование и переопределение**  
   - Создаём классы `Circle` и `Rectangle`, наследующие `Shape`.  
   ```python
   import math

   class Circle(Shape):
       def __init__(self, radius: float):
           self.radius = radius

       def area(self) -> float:
           return math.pi * self.radius ** 2

   class Rectangle(Shape):
       def __init__(self, width: float, height: float):
           self.width = width
           self.height = height

       def area(self) -> float:
           return self.width * self.height
   ```

3. **Использование `super()`**  
   - Добавляем класс `ColoredCircle`, наследующий `Circle`, и переопределяем `area`, при этом вызываем родительский метод.  
   ```python
   class ColoredCircle(Circle):
       def __init__(self, radius: float, color: str):
           super().__init__(radius)
           self.color = color

       def area(self) -> float:
           base_area = super().area()
           # Дополнительная логика (например, учёт цвета)
           return base_area
   ```

4. **Множественное наследование**  
   - Создаём класс `Square`, наследующий `Rectangle` и `ColoredCircle`.  
   ```python
   class Square(Rectangle, ColoredCircle):
       def __init__(self, side: float, color: str):
           Rectangle.__init__(self, side, side)
           ColoredCircle.__init__(self, side, color)

       def area(self) -> float:
           return Rectangle.area(self)
   ```

5. **Тестирование**  
   - Написать простые unit‑тесты с использованием `unittest`.  
   ```python
   import unittest

   class TestShapes(unittest.TestCase):
       def test_circle_area(self):
           self.assertAlmostEqual(Circle(1).area(), math.pi)

       def test_rectangle_area(self):
           self.assertEqual(Rectangle(2, 3).area(), 6)

       def test_square_area(self):
           self.assertEqual(Square(4, "red").area(), 16)

   if __name__ == "__main__":
       unittest.main()
   ```

6. **Запуск и анализ результатов**  
   - Запустить скрипт и убедиться, что все тесты проходят.  
   - Сравнить фактические площади с ожидаемыми значениями, объяснить возможные ошибки.
