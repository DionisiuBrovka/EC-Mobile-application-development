# Лабораторная работа № 20  
**Конфигурирование параметров запуска приложения**

## Цель и задачи
1. Познакомить студентов с концепцией конфигурации приложений в Python.  
2. Научить использовать модули `argparse`, `configparser` и `dotenv` для чтения параметров из командной строки, файлов конфигурации и переменных окружения.  
3. Разработать простое приложение, которое демонстрирует гибкую настройку поведения через конфигурацию.  
4. Оценить влияние различных способов конфигурации на тестирование и развёртывание.

## Теоретическая часть
| Понятие | Описание |
|--------|----------|
| Конфигурация | Набор параметров, задающих поведение программы. |
| `argparse` | Стандартный модуль для разбора аргументов командной строки. |
| `configparser` | Модуль для чтения файлов в формате INI. |
| `.env` файлы | Текстовые файлы с переменными окружения, часто используемые в разработке. |
| `dotenv` | Библиотека для загрузки переменных окружения из `.env`. |
| Приоритет конфигурации | Порядок, в котором параметры переопределяются: код → файл → переменные окружения → аргументы командной строки. |
| Паттерн «Конфигурация как объект» | Представление конфигурации в виде единого объекта (например, `dataclass`). |
| `argparse.Namespace` | Тип, используемый для хранения распарсенных аргументов. |
| `configparser.ConfigParser` | Класс, позволяющий работать с секциями и ключами INI‑файла. |
| `dotenv.load_dotenv()` | Функция, загружающая переменные из файла в окружение. |

## Ход выполнения работы
1. **Подготовка проекта**  
   Создайте каталог `lab20` и файлы `app.py`, `config.ini`, `.env`.  
   ```bash
   mkdir lab20 && cd lab20
   touch app.py config.ini .env
   ```

2. **Создание файла конфигурации**  
   В `config.ini` определите секцию `[app]` с параметрами по умолчанию.  
   ```ini
   [app]
   log_level = INFO
   max_workers = 4
   ```
3. **Создание файла переменных окружения**  
   В `.env` задайте переменные, которые могут переопределять параметры из INI.  
   ```dotenv
   LOG_LEVEL=DEBUG
   MAX_WORKERS=8
   ```
4. **Разработка вспомогательных функций**  
   В `app.py` импортируйте модули и создайте функцию `load_config()`, которая объединяет конфигурацию из всех источников.  
   ```python
   import argparse
   import configparser
   from dotenv import load_dotenv
   import os

   def load_config():
       # 1) Загрузка переменных окружения
       load_dotenv()
       
       # 2) Чтение INI‑файла
       cfg = configparser.ConfigParser()
       cfg.read('config.ini')
       
       # 3) Создание argparse
       parser = argparse.ArgumentParser()
       parser.add_argument('--log-level', help='Уровень логирования')
       parser.add_argument('--max-workers', type=int, help='Кол-во потоков')
       
       # 4) Получаем аргументы
       args = parser.parse_args()
       
       # 5) Собираем итоговую конфигурацию
       config = {
           'log_level': args.log_level or os.getenv('LOG_LEVEL') or cfg.get('app', 'log_level'),
           'max_workers': args.max_workers or int(os.getenv('MAX_WORKERS', cfg.get('app', 'max_workers')))
       }
       return config
   ```

5. **Главная логика приложения**  
   В `app.py` реализуйте простую задачу (например, генерацию чисел).  
   ```python
   def main():
       cfg = load_config()
       print(f"Запуск с логированием: {cfg['log_level']}, потоков: {cfg['max_workers']}")
       # Здесь могла бы быть реальная логика
   if __name__ == "__main__":
       main()
   ```

6. **Тестирование**  
   - Запустите без аргументов: `python app.py`.  
   - Переопределите через аргументы: `python app.py --log-level WARNING`.  
   - Переопределите через переменные окружения: `export LOG_LEVEL=ERROR && python app.py`.  
   Сравните выводы и убедитесь, что приоритеты соблюдены.

7. **Анализ результатов**  
   Составьте таблицу, в которой перечислены все источники конфигурации и их приоритет.  
   Оцените, насколько удобно менять параметры без перекомпиляции кода.
