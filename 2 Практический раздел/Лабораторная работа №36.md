# Лабораторная работа № 36  
Создание связанных служб и обработка разрешений  

## Цель и задачи  
1. Научиться создавать и связывать службы в Python‑приложениях.  
2. Овладеть механизмом управления доступом к ресурсам через разрешения.  
3. Разработать простое приложение, демонстрирующее работу служб и проверки прав.  
4. Провести экспериментальную проверку корректности работы системы разрешений.  

## Теоретическая часть  
| Понятие | Описание |
|---------|----------|
| **Служба (Service)** | Модуль, реализующий конкретную бизнес‑функцию, независимый от пользовательского интерфейса. |
| **Контейнер служб** | Система управления жизненным циклом служб (создание, инициализация, уничтожение). |
| **Разрешение (Permission)** | Флаг или объект, определяющий доступ пользователя к конкретной операции. |
| **Пользователь** | Идентификатор, обладающий набором разрешений. |
| **ACL (Access Control List)** | Список разрешений, применяемых к объекту. |
| **RBAC (Role‑Based Access Control)** | Модель контроля доступа, где роли связываются с разрешениями. |
| **Middleware** | Компонент, обрабатывающий запросы до и после службы. |
| **Dependency Injection** | Инъекция зависимостей – передача зависимостей в конструктор службы. |
| **Context‑Manager** | Контекстный менеджер (`with`) для управления ресурсами. |
| **Паттерн Singleton** | Обеспечение единственного экземпляра службы. |

## Ход выполнения работы  

1. **Создание проекта**  
   - Инициализируйте виртуальное окружение и установите необходимые пакеты.  
   ```bash
   python -m venv venv
   source venv/bin/activate
   pip install fastapi uvicorn python-dotenv
   ```  

2. **Определение модели пользователя**  
   - Создайте `models.py` с классом `User` и списком разрешений.  
   ```python
   # models.py
   from typing import List
   from pydantic import BaseModel

   class User(BaseModel):
       id: int
       username: str
       permissions: List[str]
   ```

3. **Разработка службы аутентификации**  
   - Файл `auth_service.py`: возвращает объект `User` после проверки токена.  
   ```python
   # auth_service.py
   from models import User

   def get_user_from_token(token: str) -> User:
       # В реальном приложении проверка JWT/Basic
       if token == "admin-token":
           return User(id=1, username="admin", permissions=["read", "write", "delete"])
       else:
           return User(id=2, username="guest", permissions=["read"])
   ```

4. **Создание службы ресурсов**  
   - Файл `resource_service.py`: реализует CRUD‑операции, проверяя разрешения.  
   ```python
   # resource_service.py
   from models import User
   from typing import Dict

   _store: Dict[int, str] = {}

   def read_resource(user: User, resource_id: int) -> str:
       if "read" not in user.permissions:
           raise PermissionError("Нет прав на чтение")
       return _store.get(resource_id, "Не найдено")

   def write_resource(user: User, resource_id: int, value: str) -> None:
       if "write" not in user.permissions:
           raise PermissionError("Нет прав на запись")
       _store[resource_id] = value
   ```

5. **Интеграция с FastAPI**  
   - Файл `main.py`: маршруты, middleware для извлечения пользователя из заголовка.  
   ```python
   # main.py
   from fastapi import FastAPI, Header, HTTPException
   from auth_service import get_user_from_token
   from resource_service import read_resource, write_resource

   app = FastAPI()

   @app.get("/resource/{resource_id}")
   async def get_resource(resource_id: int, token: str = Header(...)):
       user = get_user_from_token(token)
       try:
           return {"value": read_resource(user, resource_id)}
       except PermissionError as e:
           raise HTTPException(status_code=403, detail=str(e))

   @app.post("/resource/{resource_id}")
   async def set_resource(resource_id: int, value: str, token: str = Header(...)):
       user = get_user_from_token(token)
       try:
           write_resource(user, resource_id, value)
           return {"status": "ok"}
       except PermissionError as e:
           raise HTTPException(status_code=403, detail=str(e))
   ```

6. **Запуск и тестирование**  
   - Запустите сервер:  
   ```bash
   uvicorn main:app --reload
   ```  
   - Используйте `curl` или Postman, передавая заголовок `token`.  
   ```bash
   curl -H "token: admin-token" http://127.0.0.1:8000/resource/1
   curl -H "token: guest-token" -X POST -d "value=hello" http://127.0.0.1:8000/resource/1
   ```

7. **Анализ результатов**  
   - Проверьте, что пользователь без прав не может выполнить действие.  
   - Сравните поведение при разных токенах.
