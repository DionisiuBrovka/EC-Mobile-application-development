# Лабораторная работа № 39  
Соединение и взаимодействие с Firebase  

---

## Цель и задачи  

1. Познакомиться с облачной платформой Firebase и её основными сервисами.  
2. Научиться подключать проект Python к Firebase Realtime Database.  
3. Реализовать чтение и запись данных в базу через SDK.  
4. Оценить производительность и безопасность соединения.  

---

## Теоретическая часть  

| Понятие | Краткое определение |
|---------|---------------------|
| Firebase | Облачная платформа Google для мобильных и веб‑приложений. |
| Realtime Database | NoSQL‑база, поддерживающая синхронизацию данных в реальном времени. |
| SDK (Software Development Kit) | Набор библиотек, позволяющих интегрировать приложение с сервисом. |
| REST‑API | Протокол взаимодействия, основанный на HTTP‑запросах. |
| JSON | Формат обмена данными, используемый Firebase. |
| Токен авторизации | Строка, подтверждающая права доступа к базе. |
| Ключ API | Уникальный идентификатор проекта в Firebase. |
| Правила безопасности | Набор условий, ограничивающих доступ к данным. |
| Транзакция | Операция, гарантирующая атомарность записи. |
| Слушатель (listener) | Функция, реагирующая на изменения в базе. |

## Ход выполнения работы  

1. **Создание проекта в Firebase**  
   - Перейти в консоль Firebase, создать новый проект, включить Realtime Database.  
   - Скопировать `databaseURL` и `serviceAccountKey.json` (ключ сервисного аккаунта).  

2. **Установка необходимых библиотек**  
   ```bash
   pip install firebase-admin
   ```  

3. **Инициализация приложения**  
   ```python
   import firebase_admin
   from firebase_admin import credentials, db

   cred = credentials.Certificate('serviceAccountKey.json')
   firebase_admin.initialize_app(cred, {
       'databaseURL': 'https://<PROJECT_ID>.firebaseio.com'
   })
   ```

4. **Чтение данных**  
   ```python
   ref = db.reference('users')
   users = ref.get()
   print(users)
   ```

5. **Запись данных**  
   ```python
   new_user_ref = db.reference('users').push()
   new_user_ref.set({
       'name': 'Иван',
       'age': 23,
       'email': 'ivan@example.com'
   })
   ```

6. **Обновление данных**  
   ```python
   user_ref = db.reference('users/USER_KEY')
   user_ref.update({'age': 24})
   ```

7. **Удаление данных**  
   ```python
   user_ref.delete()
   ```

8. **Транзакция**  
   ```python
   def increment_counter(current):
       return (current or 0) + 1

   counter_ref = db.reference('counter')
   counter_ref.transaction(increment_counter)
   ```

9. **Слушатель изменений**  
   ```python
   def listener(event):
       print(f'Path: {event.path}, Data: {event.data}')

   ref.listen(listener)
   ```

10. **Проверка правил безопасности**  
    - В консоли Firebase перейти в «Database → Rules» и убедиться, что чтение/запись разрешено только авторизованным пользователям.

## Контрольные вопросы / задания для самопроверки

2. Как получить `serviceAccountKey.json` и зачем он нужен?  
3. В чем разница между `set()`, `update()` и `push()`?  
4. Как реализовать транзакцию в Firebase и зачем она нужна?  
5. Как настроить правила безопасности для чтения только по ключу?  
6. Какие проблемы могут возникнуть при работе с большим объёмом данных в реальном времени?  
7. Как проверить, что слушатель работает корректно?  
8. Что произойдет, если в коде не инициализировать приложение Firebase?

10. Как можно ограничить доступ к базе по IP‑адресу?
