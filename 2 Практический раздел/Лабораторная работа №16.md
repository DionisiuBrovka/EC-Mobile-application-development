# Лабораторная работа № 16  
Разработка многопоточных приложений

## Цель и задачи

1. Ознакомить студентов с основами многопоточности в Python.  
2. Научить использовать модули `threading` и `concurrent.futures`.  
3. Разработать простое многопоточное приложение, демонстрирующее синхронизацию потоков.  
4. Провести экспериментальное измерение производительности и проанализировать результаты.

## Теоретическая часть

| Понятие | Определение |
|---------|-------------|
| Поток (thread) | Самостоятельный поток выполнения внутри процесса. |
| Многопоточность | Параллельное выполнение нескольких потоков. |
| `threading.Thread` | Класс, представляющий поток, управляемый модулем `threading`. |
| `threading.Lock` | Объект синхронизации, позволяющий защитить критическую секцию. |
| `threading.Event` | Флаг, используемый для сигнализации между потоками. |
| `queue.Queue` | Блокирующая очередь, безопасная для потоков. |
| `concurrent.futures.ThreadPoolExecutor` | Высокоуровневый пул потоков. |
| GIL (Global Interpreter Lock) | Блокировка, обеспечивающая, что в любой момент времени в CPython выполняется только один поток. |
| `join()` | Метод, заставляющий основной поток ждать завершения дочернего. |
| `daemon` | Флаг потока, при котором поток завершается, когда завершаются все непарольные потоки. |

## Ход выполнения работы

1. **Подготовка окружения**  
   Установите Python 3.10+ и создайте виртуальное окружение.  
   ```bash
   python -m venv venv
   source venv/bin/activate   # Windows: venv\Scripts\activate
   pip install --upgrade pip
   ```

2. **Создание простой задачи**  
   Напишите функцию `compute` выполняющую вычисления (например, факториал) и возвращающую результат.  
   ```python
   def compute(n):
       result = 1
       for i in range(1, n+1):
           result *= i
       return result
   ```

3. **Параллельное выполнение без синхронизации**  
   Запустите несколько потоков, каждый из которых вызывает `compute`.  
   ```python
   import threading

   def worker(n, results, idx):
       results[idx] = compute(n)

   nums = [10, 15, 20, 25]
   results = [None] * len(nums)
   threads = []

   for i, n in enumerate(nums):
       t = threading.Thread(target=worker, args=(n, results, i))
       threads.append(t)
       t.start()

   for t in threads:
       t.join()

   print(results)
   ```

4. **Добавление синхронизации**  
   Используйте `threading.Lock` для защиты общей переменной.  
   ```python
   lock = threading.Lock()
   total = 0

   def worker_sum(n):
       global total
       with lock:
           total += compute(n)

   threads = [threading.Thread(target=worker_sum, args=(n,)) for n in nums]
   for t in threads:
       t.start()
   for t in threads:
       t.join()

   print('Total:', total)
   ```

5. **Использование пула потоков**  
   Перепишите задачу через `ThreadPoolExecutor`.  
   ```python
   from concurrent.futures import ThreadPoolExecutor, as_completed

   with ThreadPoolExecutor(max_workers=4) as executor:
       futures = {executor.submit(compute, n): n for n in nums}
       for future in as_completed(futures):
           n = futures[future]
           print(f'Factorial of {n} = {future.result()}')
   ```

6. **Экспериментальная часть**  
   - Измерьте время выполнения при разном количестве потоков (1, 2, 4, 8).  
   - Сравните с последовательным вариантом.  
   ```python
   import time

   def measure(n_workers):
       with ThreadPoolExecutor(max_workers=n_workers) as executor:
           start = time.perf_counter()
           futures = [executor.submit(compute, n) for n in nums]
           for f in futures:
               f.result()
           end = time.perf_counter()
       return end - start

   for workers in [1, 2, 4, 8]:
       print(f'{workers} workers: {measure(workers):.4f}s')
   ```

7. **Анализ результатов**  
   Составьте таблицу со временем выполнения, обсудите влияние GIL и ограничений.
