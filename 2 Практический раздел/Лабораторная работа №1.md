# Лабораторная работа № 1  
Создание, компилирование, отладка и выполнение проектов в интегрированной среде разработки

## Цель и задачи
1. Ознакомить студентов с основными возможностями IDE для Python.  
2. Научить создавать, сохранять и управлять проектами в IDE.  
3. Разобрать этапы компиляции, отладки и запуска кода.  
4. Развить навыки диагностики ошибок и анализа результатов выполнения программы.

## Теоретическая часть
1. **IDE (Integrated Development Environment)** – комплекс программных средств, объединяющий редактор кода, компилятор/интерпретатор, отладчик, менеджер зависимостей и др.  
2. **Скрипт** – программа, написанная на языке высокого уровня, обычно выполняется без явной компиляции.  
3. **Модуль** – файл Python, содержащий функции, классы и переменные, который можно импортировать в другие скрипты.  
4. **Пакет** – каталог с модулем `__init__.py`, позволяющий сгруппировать связанные модули.  
5. **Система управления версиями** – инструменты (Git, SVN) для контроля изменений в коде.  
6. **Отладчик** – средство, позволяющее пошагово выполнять программу, ставить точки останова и просматривать состояние переменных.  
7. **Точка останова (breakpoint)** – инструкция `breakpoint()` или кнопка в IDE, останавливающая исполнение в заданном месте.  
8. **Стек вызовов** – последовательность активных вызовов функций, отображаемая в отладчике.  
9. **Логирование** – запись событий программы в файл или консоль с помощью модуля `logging`.  
10. **Пакет `venv`** – инструмент для создания изолированных виртуальных сред.

## Ход выполнения работы
1. **Создание проекта**  
   - Открыть IDE (PyCharm, VS Code, Rider и т.д.).  
   - Выбрать `File → New Project`.  
   - Ввести имя проекта `lab1_project`, указать расположение, выбрать интерпретатор Python.  
   - Создать виртуальную среду `venv` автоматически.  

2. **Добавление модулей**  
   - В каталоге проекта создать файл `main.py`.  
   - Вставить простую функцию:  
     ```python
     def greet(name: str) -> str:
         return f"Привет, {name}!"
     ```  
   - Создать файл `utils.py` с вспомогательной функцией:  
     ```python
     def add(a: int, b: int) -> int:
         return a + b
     ```

3. **Подключение модулей**  
   - В `main.py` импортировать функции:  
     ```python
     from utils import add

     if __name__ == "__main__":
         print(greet("Алексей"))
         print("Сумма:", add(3, 5))
     ```

4. **Запуск без отладки**  
   - Нажать кнопку «Run» (обычно зелёный треугольник).  
   - Проверить вывод в консоли.  

5. **Установка точки останова**  
   - Кликнуть слева от строки `print(greet("Алексей"))`.  
   - Точка останова станет красной.  

6. **Запуск отладки**  
   - Нажать кнопку «Debug».  
   - Программа остановится на точке останова.  
   - В окне отладчика просмотреть переменные `name`, `greet`, `add`.  

7. **Пошаговое исполнение**  
   - Использовать кнопки «Step Over» (F8) и «Step Into» (F7).  
   - Наблюдать изменение значений переменных.  

8. **Исправление ошибки**  
   - Внести в `utils.py` ошибку: `return a - b`.  
   - Запустить отладку, обнаружить неверный результат, исправить и повторить.  

9. **Логирование**  
   - Добавить в `main.py`:  
     ```python
     import logging
     logging.basicConfig(level=logging.INFO)
     logging.info("Запуск программы")
     ```  
   - Проверить появление записи в консоли.  

10. **Сохранение и экспорт проекта**  
    - Сохранить все файлы.  
    - Создать архив проекта (правый клик → Export → Zip).  
    - Отправить в лабораторный модуль.  

## Контрольные вопросы / задания для самопроверки
1. Что такое виртуальная среда и зачем она нужна?  
2. Как установить точку останова в PyCharm?  
3. Чем отличается `Step Over` от `Step Into`?  
4. Как подключить модуль `utils.py` к `main.py`?  
5. Что выводит `logging.info("Запуск программы")` при уровне `INFO`?  
6. Какие ошибки могут возникнуть при отсутствии виртуальной среды?  
7. Как экспортировать проект в виде архива?  
8. Что такое стек вызовов и как его использовать в отладчике?  
9. Как проверить, что функция `add` работает корректно после исправления?  
10. Что произойдет, если в `main.py` не будет `if __name__ == "__main__":`?
