# Лабораторная работа № 26  
Создание модульной структуры приложения  

## Цель и задачи  
1. Научиться проектировать и реализовывать модульную структуру Python‑приложения.  
2. Освоить принципы разделения кода на пакеты и модули.  
3. Понять роль файлов `__init__.py` и их влияние на импорт.  
4. Разобрать шаблоны проектирования, применяемые при создании библиотек.  

## Теоретическая часть  
- **Модуль** – файл с расширением `.py`, содержащий код, который можно импортировать.  
- **Пакет** – каталог, содержащий файл `__init__.py` и один или несколько модулей.  
- **`__init__.py`** – специальный файл, делающий каталог импортируемым, может инициализировать пакет.  
- **Импорт** – процесс приведения кода модуля в вид, доступный в текущем пространстве имён (`import`, `from ... import ...`).  
- **Relative import** – импорт, основанный на структуре пакета (`from . import module`).  
- **Namespace package** – пакет без файла `__init__.py`, поддерживаемый PEP 420.  
- **`setup.py` / `pyproject.toml`** – файлы метаданных для сборки и публикации пакета.  
- **`requirements.txt`** – список зависимостей проекта.  
- **PEP 8** – стиль кодирования Python, рекомендуемый для читаемости.  
- **`__all__`** – список публичных объектов модуля, который попадает в `from module import *`.  
- **`__main__`** – блок, исполняемый при запуске модуля как скрипт.  

## Ход выполнения работы  

1. **Создать структуру каталогов**  
   ```text
   my_app/
       src/
           __init__.py
           core/
               __init__.py
               utils.py
               processor.py
           cli/
               __init__.py
               main.py
       tests/
           __init__.py
           test_processor.py
       README.md
       setup.py
       requirements.txt
   ```  
   *Пояснение:* `src` содержит исходный код, `tests` – тесты, корень – файлы проекта.  

2. **Заполнить `__init__.py` в корневом пакете**  
   ```python
   # src/__init__.py
   """Главный пакет my_app."""
   __all__ = ['core', 'cli']
   ```  
   *Пояснение:* Указывает, какие подпакеты доступны при импорте `my_app`.  

3. **Разработать модуль `utils.py`**  
   ```python
   # src/core/utils.py
   def greet(name: str) -> str:
       """Возвращает строку приветствия."""
       return f"Hello, {name}!"
   ```  
   *Пояснение:* Простая функция для демонстрации импортов.  

4. **Создать модуль `processor.py`**  
   ```python
   # src/core/processor.py
   from .utils import greet

   def process(names):
       """Обрабатывает список имён и выводит приветствия."""
       return [greet(n) for n in names]
   ```  
   *Пояснение:* Использует относительный импорт из того же пакета.  

5. **Разработать точку входа `main.py`**  
   ```python
   # src/cli/main.py
   import argparse
   from my_app.core.processor import process

   def main():
       parser = argparse.ArgumentParser(description="Demo app")
       parser.add_argument('names', nargs='+', help='Names to greet')
       args = parser.parse_args()
       for msg in process(args.names):
           print(msg)

   if __name__ == "__main__":
       main()
   ```  
   *Пояснение:* Пример CLI‑приложения, использующего пакет.  

6. **Написать простые тесты**  
   ```python
   # tests/test_processor.py
   import unittest
   from my_app.core.processor import process

   class TestProcessor(unittest.TestCase):
       def test_process(self):
           self.assertEqual(process(['Alice', 'Bob']),
                            ['Hello, Alice!', 'Hello, Bob!'])

   if __name__ == '__main__':
       unittest.main()
   ```  
   *Пояснение:* Проверка корректности функции `process`.  

7. **Сконфигурировать `setup.py`**  
   ```python
   # setup.py
   from setuptools import setup, find_packages

   setup(
       name='my_app',
       version='0.1.0',
       packages=find_packages('src'),
       package_dir={'': 'src'},
       python_requires='>=3.8',
       install_requires=[],
   )
   ```  
   *Пояснение:* Позволяет установить пакет локально (`pip install -e .`).  

8. **Запустить тесты**  
   ```bash
   python -m unittest discover tests
   ```  
   *Пояснение:* Проверка, что модуль работает независимо.  

9. **Запустить приложение**  
   ```bash
   python -m my_app.cli.main Alice Bob
   ```  
   *Результат:*  
   ```text
   Hello, Alice!
   Hello, Bob!
   ```
