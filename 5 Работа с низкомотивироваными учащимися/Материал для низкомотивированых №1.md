# Обработка исключительных ситуаций

## Цель и задачи занятия  
- Понять, что такое исключения и зачем их обрабатывать.  
- Научиться пользоваться конструкциями `try…except`, `else`, `finally`.  
- Освоить принципы логирования ошибок и создание собственных исключений.  

## Почему это важно?  
В реальных проектах ошибки неизбежны: пользователь может ввести неверные данные, файл может отсутствовать, сервер может быть недоступен. Правильная обработка исключений позволяет программе «выжить» и дать пользователю понятное сообщение, а разработчику – быстро отладить проблему.

## Основные понятия и идеи  

| Термин | Что это такое | Пример |
|--------|---------------|--------|
| Исключение | Специальный объект, сигнализирующий об ошибке | `ZeroDivisionError` |
| `try` | Блок кода, который может вызвать исключение | `try: x = 1 / 0` |
| `except` | Перехватывает конкретный тип исключения | `except ZeroDivisionError:` |
| `else` | Код, выполняемый, если исключения не возникло | `else: print('OK')` |
| `finally` | Код, выполняемый всегда, даже если ошибка | `finally: print('Done')` |
| Пользовательское исключение | Свой класс ошибки, наследуемый от `Exception` | `class MyError(Exception): pass` |
| Логирование | Запись ошибок в файл/консоль для последующего анализа | `logging.error(...)` |

## Объяснение темы  

### 1. [10 мин] Что такое исключения?  
Исключения – это как сигнал «попробуй снова».  
> *Аналогия*: если вы пытаетесь открыть дверь, но она заперта, вы слышите сигнал, что нужно ввести пароль.  

```python
# Простой пример
print("Начинаем")
1 / 0  # Ошибка! Программа падает
```

### 2. [15 мин] Структура `try…except`  
```python
try:
    # рискованный код
    x = int(input("Введите число: "))
except ValueError:
    print("Нужно было ввести число, а не текст!")
```

> Попробуй сам!  
> Что произойдёт, если вместо `ValueError` написать `TypeError`?  

### 3. [10 мин] `else` и `finally`  
```python
try:
    f = open('file.txt')
except FileNotFoundError:
    print('Файл не найден')
else:
    print('Файл успешно открыт')
    f.close()
finally:
    print('Конец работы с файлом')
```

### 4. [15 мин] Создание собственных исключений  
```python
class NegativeAgeError(Exception):
    pass

def set_age(age):
    if age < 0:
        raise NegativeAgeError("Возраст не может быть отрицательным")
    return age
```

**Мини‑задание**  
> Добавь проверку, чтобы возраст был не больше 120.  

### 5. [10 мин] Логирование ошибок  
```python
import logging
logging.basicConfig(filename='app.log', level=logging.ERROR)

try:
    1 / 0
except ZeroDivisionError as e:
    logging.error("Ошибка деления на ноль: %s", e)
```

### 6. [10 мин] Практическая задача: калькулятор с безопасным вводом  
```python
def safe_div(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        return "Нельзя делить на ноль"

print(safe_div(10, 0))
```

**Мини‑задание**  
> Добавь обработку `TypeError`, чтобы пользователь вводил только числа.  

## Итог и выводы  
- Исключения позволяют программе «выжить» после ошибки.  
- Конструкции `try…except`, `else`, `finally` дают контроль над потоком выполнения.  
- Создавая собственные ошибки, вы делаете код более читаемым и удобным для отладки.  
- Логирование фиксирует детали ошибок для последующего анализа.
