# Методы и конструкторы

## Цель и задачи занятия  
- Понять, что такое методы и конструкторы в Python.  
- Научиться определять и вызывать методы у объектов.  
- Знать, как писать собственные конструкторы (`__init__`).  
- Увидеть, как методы помогают упорядочить код и сделать его более читабельным.  

## Почему это важно?  
В реальной работе программистов объекты часто содержат много данных и действий.  
Методы позволяют логически группировать действия, а конструкторы автоматически
инициализируют объекты. Это делает ваш код гибким, расширяемым и менее подверженным ошибкам.  

## Основные понятия и идеи  
| Термин | Что это такое? |
|--------|----------------|
| **Класс** | Шаблон, по которому создаются объекты. |
| **Объект** | Конкретный экземпляр класса с собственными данными. |
| **Метод** | Функция, объявленная внутри класса и вызываемая через объект. |
| **`self`** | Ссылка на сам объект, через которую можно обращаться к его атрибутам. |
| **Конструктор** (`__init__`) | Специальный метод, запускаемый при создании объекта. |
| **Атрибут** | Переменная, принадлежащая объекту. |
| **Модификатор доступа** | Ограничение видимости атрибутов и методов (например, `__private`). |

## Объяснение темы  

### 1. Введение в методы [10 мин]  
- **Идея**: методы — это как кнопки на устройстве: нажимаешь, и что‑то происходит.  
- **Пример**:  
  ```python
  class Car:
      def __init__(self, brand):
          self.brand = brand

      def honk(self):
          print("Бип! Бип!")

  my_car = Car("Toyota")
  my_car.honk()   # выводит: Бип! Бип!
  ```
- **Пояснение**: `honk` — метод, `self` позволяет нам обращаться к конкретному автомобилю.

### 2. Параметры методов [15 мин]  
- Методы могут принимать аргументы, помимо `self`.  
- **Пример**:  
  ```python
  class Calculator:
      def add(self, a, b):
          return a + b

  calc = Calculator()
  print(calc.add(3, 5))  # 8
  ```
- **Игра**: представьте, что `add` — это рецепт: вы берете два ингредиента и получаете результат.  

### 3. Конструкторы и инициализация [20 мин]  
- Конструктор `__init__` запускается автоматически.  
- В нем можно задавать начальные значения атрибутов и выполнять проверки.  
- **Пример**:  
  ```python
  class Person:
      def __init__(self, name, age=0):
          if age < 0:
              raise ValueError("Возраст не может быть отрицательным")
          self.name = name
          self.age = age

      def greet(self):
          print(f"Привет, меня зовут {self.name}!")

  p = Person("Ирина", 25)
  p.greet()  # Привет, меня зовут Ирина!
  ```
- **Подсказка**: используйте конструктор, чтобы сразу задать «правильные» данные.

### 4. Инкапсуляция и модификаторы доступа [15 мин]  
- Атрибуты, начинающиеся с двойного подчеркивания (`__`), считаются приватными.  
- **Пример**:  
  ```python
  class BankAccount:
      def __init__(self, balance):
          self.__balance = balance

      def deposit(self, amount):
          self.__balance += amount

      def get_balance(self):
          return self.__balance
  ```
- **Пояснение**: приватные атрибуты защищены от случайного изменения извне.

### 5. Практика: создаём собственный класс [10 мин]

- Добавить методы: `read_page(page)` (выводит сообщение о прочтении) и `summary()` (выводит краткую информацию).

## Итог и выводы  
Методы и конструкторы позволяют создавать логически завершённые блоки кода, которые легко использовать и расширять. Они делают ваш код понятным и защищённым от ошибок. Практика с собственными классами закрепит навыки и повысит уверенность.
