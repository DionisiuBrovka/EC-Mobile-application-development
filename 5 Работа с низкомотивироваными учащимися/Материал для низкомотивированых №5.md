# Многопоточное программирование при работе с файлами

## Цель и задачи занятия
- Понять, почему многопоточность полезна при чтении/записи больших файлов.  
- Научиться использовать потоковый модуль `threading` и блокировки.  
- Освоить простые паттерны: поток‑производитель/потребитель.  
- Приобрести уверенность в написании кода, который работает без ошибок гонок.

## Почему это важно?
В реальной работе часто приходится обрабатывать десятки мегабайт данных: лог‑файлы, CSV‑таблицы, медиа‑файлы.  
Однопоточный подход «ждёт» завершения операции, а многопоточность позволяет:  
- экономить время,  
- использовать все ядра процессора,  
- не «зависать» пользовательский интерфейс.

## Основные понятия и идеи
| Термин | Что это такое |
|--------|---------------|
| Поток (Thread) | Самостоятельный поток выполнения внутри одного процесса. |
| Блокировка (Lock) | Механизм, запрещающий нескольким потокам одновременно менять одни и те же данные. |
| Производитель (Producer) | Поток, который создаёт данные (например, читает файл). |
| Потребитель (Consumer) | Поток, который использует данные (например, обрабатывает строки). |
| Очередь (Queue) | Безопасный для потоков контейнер, где производитель кладёт, а потребитель берёт элементы. |
| Гонка данных (Race Condition) | Ошибка, когда несколько потоков читают/пишут в одну переменную без синхронизации. |
| Гарантия завершения (Join) | Ожидание завершения всех потоков перед выходом из программы. |

## Объяснение темы

### [10 мин] Введение в `threading`
```python
import threading

def hello():
    print("Привет из потока!")

t = threading.Thread(target=hello)
t.start()
t.join()
```
*Что происходит?*  
Поток `t` запускается параллельно основному потоку, но `join()` заставляет основной ждать его окончания.

**Мини‑задача**  
> Попробуй изменить `print("Привет из потока!")` на `print("Число:", 5*5)`. Что выведет программа?

### [15 мин] Пример: чтение большого файла в фоне
```python
def read_file(path, queue):
    with open(path, 'r') as f:
        for line in f:
            queue.put(line)
    queue.put(None)  # сигнал завершения

def process_lines(queue):
    while True:
        line = queue.get()
        if line is None:
            break
        # простая обработка
        print(line.strip().upper())
```
*Почему так?*  
Поток чтения не блокирует поток обработки, а очередь гарантирует, что каждая строка будет обработана ровно один раз.

**Мини‑задача**  
> Добавь `time.sleep(0.01)` в `read_file`. Как изменится поведение?

### [10 мин] Синхронизация доступа к общим ресурсам
```python
lock = threading.Lock()
shared_counter = 0

def increment():
    global shared_counter
    for _ in range(1000):
        with lock:
            shared_counter += 1
```
*Что происходит без `lock`?*  
Поток может «пробежать» через чтение и запись, и результат будет непредсказуемый.

**Мини‑задача**  
> Удали `with lock:` и запусти программу 5 раз. Как меняется значение `shared_counter`?

### [10 мин] Паттерн «производитель‑потребитель» с `queue.Queue`
```python
from queue import Queue

q = Queue(maxsize=10)  # ограничиваем размер очереди

def producer():
    for i in range(20):
        q.put(i)
    q.put(None)

def consumer():
    while True:
        item = q.get()
        if item is None:
            break
        print(f'Потребил {item}')

threads = [threading.Thread(target=producer),
           threading.Thread(target=consumer)]
for t in threads:
    t.start()
for t in threads:
    t.join()
```
*Почему `maxsize`?*  
Ограничивает память и заставляет производитель ждать, если потребитель не успевает обрабатывать.

**Мини‑задача**  
> Установи `maxsize=1`. Что изменится в выводе?

## Итог и выводы
Многопоточность при работе с файлами позволяет ускорить обработку, не «загружать» основной поток.  
Ключ к успеху – правильное разделение задач (производитель/потребитель) и синхронизация через очереди и блокировки.  
Постепенно переходя от простого примера к реальным задачам, вы сможете уверенно использовать `threading` в своих проектах.
