# Компонент RecyclerView и выдвижные панели

## Цель и задачи занятия  
- Понять, зачем нужен `RecyclerView` и как он работает.  
- Научиться создавать простую выдвижную панель (Drawer).  
- Практически применить знания, создав небольшое приложение.  
- Увеличить уверенность в работе с UI‑компонентами.  

## Почему это важно?  
В современных мобильных приложениях почти всё отображается списком: новости, сообщения, контакты. `RecyclerView` позволяет быстро и эффективно показывать большие списки, экономя память и ресурсы. Выдвижная панель — привычный способ организации навигации, который пользователи ожидают в любом приложении.  

## Основные понятия и идеи  
1. **Adapter** – посредник, который берёт данные и превращает их в элементы списка.  
2. **ViewHolder** – контейнер для одного элемента списка, который повторно используется.  
3. **LayoutManager** – отвечает за расположение элементов (Linear, Grid, Staggered).  
4. **DiffUtil** – инструмент, который умеет находить отличия между списками и обновлять только изменённые элементы.  
5. **DrawerLayout** – контейнер, который «выдвигается» со стороны экрана и содержит навигационное меню.  
6. **NavigationView** – виджет, который обычно помещается внутри DrawerLayout и содержит пункты меню.  
7. **MenuItem** – отдельный пункт, который может открывать новый экран или выполнять действие.  

## Объяснение темы  

### 1. Что такое `RecyclerView`? [10 мин]  
`RecyclerView` — это “многоуровневый список”, который переиспользует уже созданные элементы, чтобы не создавать их заново каждый раз. Это как если бы в библиотеке читали книги: вместо того чтобы держать все книги в руках, берут одну, читают, кладут обратно и берут следующую.

#### Мини‑пример кода  
```kotlin
class MyAdapter(private val items: List<String>) : RecyclerView.Adapter<MyViewHolder>() {
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int) =
        MyViewHolder(LayoutInflater.from(parent.context).inflate(R.layout.item, parent, false))

    override fun onBindViewHolder(holder: MyViewHolder, position: Int) {
        holder.textView.text = items[position]
    }

    override fun getItemCount() = items.size
}
```

### 2. Как подключить `RecyclerView` к Activity? [15 мин]  
- Добавляем элемент в layout.  
- Инициализируем `Adapter` и `LayoutManager`.  
- Привязываем к `RecyclerView`.  

#### Мини‑пример  
```xml
<androidx.recyclerview.widget.RecyclerView
    android:id="@+id/recyclerView"
    android:layout_width="match_parent"
    android:layout_height="match_parent"/>
```

```kotlin
val recyclerView: RecyclerView = findViewById(R.id.recyclerView)
recyclerView.layoutManager = LinearLayoutManager(this)
recyclerView.adapter = MyAdapter(listOf("One", "Two", "Three"))
```

### 3. Выдвижная панель (Drawer) – как навигация в игре [15 мин]  
- `DrawerLayout` – главный контейнер, который оборачивает всё содержимое и меню.  
- `NavigationView` – меню, которое появляется при «выдвижении».  

#### Мини‑пример  
```xml
<androidx.drawerlayout.widget.DrawerLayout
    android:id="@+id/drawerLayout"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <!-- Основной контент -->
    <FrameLayout
        android:id="@+id/contentFrame"
        android:layout_width="match_parent"
        android:layout_height="match_parent"/>

    <!-- Меню -->
    <com.google.android.material.navigation.NavigationView
        android:id="@+id/navigationView"
        android:layout_width="wrap_content"
        android:layout_height="match_parent"
        android:layout_gravity="start"
        app:menu="@menu/drawer_menu"/>
</androidx.drawerlayout.widget.DrawerLayout>
```

### 4. Объединяем всё: приложение “Список + Drawer” [20 мин]  
- Создаём Activity с `DrawerLayout`.  
- Внутри добавляем `RecyclerView`.  
- Настраиваем `NavigationView` – добавляем пункты меню.  
- Реагируем на выбор пункта: меняем заголовок, фильтруем список.  

#### Мини‑пример кода  
```kotlin
navigationView.setNavigationItemSelectedListener { menuItem ->
    when (menuItem.itemId) {
        R.id.menu_all -> adapter.update(listOf("One","Two","Three"))
        R.id.menu_even -> adapter.update(listOf("Two"))
    }
    drawerLayout.closeDrawers()
    true
}
```

### 5. Оптимизация и лучшие практики [10 мин]  
- Использовать `DiffUtil` вместо `notifyDataSetChanged()`.  
- Кэшировать `ViewHolder` с `itemView.tag`.  
- Следить за размером списка: если > 50‑100 элементов, включать `setHasFixedSize(true)`.
