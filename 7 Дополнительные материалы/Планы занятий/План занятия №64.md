# Анализ ОКР №2. Firebase. Виджеты главного экрана

## Цель занятия
1. Ознакомить студентов с принципами работы Firebase как облачного бекенда для Python‑приложений.  
2. Научить проектировать и реализовывать виджеты главного экрана, взаимодействующие с Firebase.

## Задачи занятия
- Рассмотреть архитектуру Firebase и основные сервисы (Authentication, Realtime Database, Firestore).  
- Изучить методы аутентификации и чтения/записи данных из Python.  
- Разработать прототип виджета главного экрана, отображающий данные из Firebase.  
- Провести практическое закрепление через мини‑проект.  
- Оценить понимание ключевых концепций через контрольные вопросы.

## План занятия (90 минут)

- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Введение в Firebase и его API для Python — 15 мин  
- Создание и настройка проекта Firebase — 10 мин  
- Разработка виджета главного экрана (UI + логика) — 20 мин  
- Тестирование и отладка взаимодействия с Firebase — 10 мин  
- Итог занятия и обсуждение результатов — 10 мин  

## Содержание этапов

- **Организационный момент** – проверка присутствия, объявление тематики, распределение ролей при работе в парах.  
- **Актуализация знаний** – краткое обсуждение предыдущего занятия о Python‑инструментах; выяснение, какие облачные сервисы уже знакомы студентам.  
- **Введение в Firebase и его API для Python** – обзор ключевых компонентов, объяснение, как Python‑приложение подключается к Firebase через `firebase-admin`.  
- **Создание и настройка проекта Firebase** – пошаговое создание проекта в консоли Firebase, генерация сервисного ключа, подключение к проекту из IDE.  
- **Разработка виджета главного экрана** – проектирование простого UI (Tkinter/Qt), написание функций чтения/записи, интеграция с Firebase, вывод списка задач.  
- **Тестирование и отладка** – проверка корректности аутентификации, чтения данных, обработки ошибок; обсуждение найденных багов.  
- **Итог занятия** – обобщение полученных знаний, ответы на вопросы, раздача чек‑листов для самопроверки.  

## Оборудование и материалы
- Личные компьютеры с установленной IDE (PyCharm/VS Code).  
- Проектор и ноутбук преподавателя.  
- Доступ к интернету и аккаунт Google.  
- Учебник «Инструментальное программное обеспечение (изучение Python)» – раздел о работе с облачными сервисами.  
- Ссылка на официальную документацию Firebase для Python.  

## Контроль и оценивание
- Вопросы устного опроса: 3–4 вопроса о принципах аутентификации и структуре данных.  
- Мини‑задание: в течение занятия написать функцию, которая добавляет новую запись в Firestore и отображает её на виджете.  
- Краткий тест (5 вопросов) в конце занятия, проверяющий понимание ключевых понятий.  

## Домашнее задание
- Реализовать полноценный виджет «Панель задач», который позволяет:  
  1. Добавлять, редактировать и удалять задачи.  
  2. Синхронизировать данные с Firebase Firestore в реальном времени.  
- Подготовить краткий отчёт (1‑2 страницы) о проделанной работе, включая код и скриншоты результата.
