# Лабораторная работа № 8  
Разработка программ, реализующих механизм наследования  

## Цель занятия  
1. Научить студентов применять наследование при проектировании и реализации классов в Python.  
2. Познакомить с практическими аспектами переопределения методов и использования `super()`.

## Задачи занятия  
1. Выявить основные принципы объектно‑ориентированного программирования, связанные с наследованием.  
2. Реализовать простую иерархию классов, демонстрирующую наследование и переопределение.  
3. Применить `super()` для вызова методов базового класса.  
4. Провести рефакторинг кода для повышения его читаемости и расширяемости.  
5. Оценить полученные решения по заданным критериям.

## План занятия (90 мин)  

- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Изучение нового материала — 25 мин  
- Практическая работа (параллельно) — 20 мин  
- Закрепление — 10 мин  
- Итог занятия — 10 мин  

## Содержание этапов  

- **Организационный момент** – проверка присутствия, объявление темы и целей, краткое разъяснение правил работы с IDE.  
- **Актуализация знаний** – короткий викторинный вопрос по наследованию, обсуждение примеров из предыдущих лабораторий.  
- **Изучение нового материала** – лекционная часть: основные понятия (`class`, `__init__`, `super()`, множественное наследование), примеры кода, потенциальные ошибки.  
- **Практическая работа** – студенты реализуют иерархию классов: `Vehicle → Car → SportsCar`. Переопределяют методы, используют `super()`, тестируют работу.  
- **Закрепление** – разбор решений, групповая дискуссия о возможных улучшениях, ответы на вопросы.  
- **Итог занятия** – подведение итогов, раздача чек‑листов для самопроверки, объявление домашнего задания.

## Оборудование и материалы  

- 2 ПК (или ноутбука) с установленным Python 3.10+ и IDE (PyCharm, VS Code).  
- Проектор/дисплей для демонстрации кода.  
- Копии лабораторных инструкций (PDF/Markdown).  
- Таблица критериев оценивания (раздаточный материал).  

## Контроль и оценивание  

- **Краткие вопросы** по каждому этапу (проверка понимания).  
- **Тест** из 5 вопросов с открытым ответом (проверка теоретических знаний).  
- **Практическое задание**: оценка кода по критериям: корректность, использование `super()`, читаемость, комментарии.  

## Домашнее задание  

1. Создать иерархию классов `Animal → Mammal → Dog`, реализовать методы `speak()` и `move()`, переопределить их в каждом классе.  
2. Написать краткий доклад (200‑300 слов) о преимуществах и недостатках множественного наследования в Python.
