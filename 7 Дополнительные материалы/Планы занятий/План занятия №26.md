# Лабораторная работа № 15 Поточная модель

## Цель занятия  
1. Научить студентов моделировать потоковые данные с использованием Python, применяя библиотеки `asyncio`, `aiohttp` и `pandas`.  
2. Развить навыки проектирования и реализации асинхронных пайплайнов для обработки больших потоков информации.

## Задачи занятия  
- Ознакомить с концепцией потоковой обработки данных и её отличиями от пакетной.  
- Познакомить с инструментами асинхронного программирования в Python.  
- Практически реализовать простую потоковую модель: чтение, трансформация и запись данных.  
- Оценить эффективность реализации и выявить возможные узкие места.  

## План занятия  
- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Теоретический ввод в потоковую модель — 15 мин  
- Демонстрация готового примера — 10 мин  
- Практическая работа (раздел 1) — 15 мин  
- Практическая работа (раздел 2) — 15 мин  
- Анализ результатов и обсуждение — 10 мин  
- Итог занятия — 5 мин  

## Содержание этапов  
- **Организационный момент** – проверка присутствия, раздача материалов, настройка рабочих станций.  
- **Актуализация знаний** – быстрый опрос студентов о предыдущих занятиях по асинхронности и работе с потоками.  
- **Теоретический ввод в потоковую модель** – объяснение терминов «поток данных», «потоковая модель», «парадигма событий» и обзор основных библиотек.  
- **Демонстрация готового примера** – преподаватель показывает полностью рабочий скрипт, объясняя ключевые моменты кода.  
- **Практическая работа (раздел 1)** – студенты создают асинхронный генератор, считывающий данные из CSV в режиме реального времени.  
- **Практическая работа (раздел 2)** – добавление этапов трансформации (фильтрация, агрегация) и сохранения результатов в базу SQLite.  
- **Анализ результатов и обсуждение** – проверка корректности вывода, оценка времени выполнения, обсуждение возможных улучшений.  
- **Итог занятия** – подведение итогов, раздача чек-листа по оценке качества кода, напоминание о домашнем задании.  

## Оборудование и материалы  
- Лаптопы студентов с предустановленным Python 3.10+, VS Code и расширением Pylance.  
- Проектор и экран для демонстрации кода.  
- Сервер SQLite на каждом компьютере.  
- Входные данные: `stream_data.csv` (пример 100 000 строк).  
- Учебник: *Python для анализа данных*, глава 12.  

## Контроль и оценивание  
- Краткие вопросы в ходе занятия (5 мин).  
- Тестовый чек‑лист выполнения задания (5 мин).  
- Оценка кода по критериям: читаемость, асинхронность, эффективность, обработка ошибок (30 мин).  

## Домашнее задание  
1. Реализовать потоковую модель, которая читает данные из API (использовать `aiohttp`), обрабатывает их и сохраняет в PostgreSQL.  
2. Написать короткий отчёт (не более 3 строк) о производительности вашего решения по сравнению с пакетным подходом.
