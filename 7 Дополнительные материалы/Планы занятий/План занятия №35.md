# Лабораторная работа № 20: Конфигурирование параметров запуска приложения

## Цель занятия
- Научить студентов формировать и применять конфигурационные файлы для запуска Python‑приложений.  
- Развить навыки работы с библиотеками `argparse`, `configparser` и `json` для динамической настройки параметров.

## Задачи занятия
1. Ознакомиться с принципами конфигурации программного обеспечения.  
2. Освоить синтаксис и использование `argparse` для командной строки.  
3. Практически применить `configparser` и `json` для чтения конфигурационных файлов.  
4. Создать собственное приложение, способное читать параметры из разных источников.  
5. Проверить корректность работы конфигурации в условиях реальных задач.

## План занятия (90 мин)

- Организационный момент — 5 мин  
- Актуализация знаний — 10 мин  
- Изучение нового материала — 25 мин  
- Практическая работа — 35 мин  
- Закрепление — 10 мин  
- Итог занятия — 5 мин  

### Содержание этапов

- **Организационный момент**  
  Приветствие, проверка присутствия, раздача списков задач, установка рабочих сред.  

- **Актуализация знаний**  
  Быстрый опрос по ранее изученным темам: работа с переменными окружения, простые скрипты и модули.  

- **Изучение нового материала**  
  Теоретическая часть: принципы конфигурации, сравнение методов (`argparse`, `configparser`, `json`), примеры кода, обсуждение преимуществ и ограничений каждого подхода.  

- **Практическая работа**  
  Студенты создают небольшое приложение, которое:
  1. Принимает аргументы командной строки (путь к файлу конфигурации, логуровый уровень).  
  2. Загружает настройки из указанного файла (`ini` или `json`).  
  3. Выводит итоговые параметры в консоль.  
  Преподаватель обеспечивает поддержку, отвечает на вопросы.  

- **Закрепление**  
  Групповая проверка кода, обсуждение возникающих проблем, обмен лучшими практиками.  

- **Итог занятия**  
  Подведение итогов, разъяснение домашнего задания, ответы на вопросы.  

## Оборудование и материалы
- Персональные компьютеры со стандартной установкой Python 3.10+  
- IDE (PyCharm, VSCode или аналог)  
- Проектор и экран для демонстрации кода  
- Учебник «Инструментальное программное обеспечение (изучение Python)» – главы 12–14  
- Печатные раздатки с примерами конфигурационных файлов  
- Интернет‑подключение для доступа к репозиториям и документации  

## Контроль и оценивание
- **Краткий тест** (5 вопросов) в конце занятия по темам `argparse`, `configparser`, `json`.  
- **Код‑ревью**: преподаватель проверит, насколько корректно студент использовал аргументы и конфигурационные файлы.  
- **Рефлексия**: студент пишет 3‑4 предложения о том, какие сложности возникли и как они были решены.  

## Домашнее задание
1. Создать приложение, которое читает конфигурацию из файла `config.yaml` (использовать библиотеку `PyYAML`) и выводит параметры в виде таблицы.  
2. Написать скрипт, который сравнивает два конфигурационных файла и выводит разницу.  
Устные ответы и код будут оценены по критериям: корректность, читаемость, соблюдение принципов DRY.
