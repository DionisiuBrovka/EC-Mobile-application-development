# Многопоточное программирование при работе с файлами

## Цель занятия
- Изучить принципы многопоточного доступа к файловой системе в Python и оценить их влияние на производительность и безопасность данных.
- Научиться применять модули `threading` и `concurrent.futures` для реализации параллельной обработки файлов.

## Задачи занятия
1. Ознакомиться с особенностями многопоточного доступа к файлам.
2. Понять роль блокировок и синхронизации потоков.
3. Научиться использовать `ThreadPoolExecutor` для параллельной записи и чтения.
4. Оценить потенциальные проблемы (race conditions, deadlock) и способы их предотвращения.

- Организационный момент — 5 мин

## Содержание этапов
- **Организационный момент**  
  Приветствие, проверка присутствия, краткое объявление темы и целей занятия.

- **Актуализация знаний**  
  Быстрый опрос: какие примеры многопоточного кода уже знакомы, какие проблемы встречались при работе с файлами.

- **Введение в многопоточное файловое I/O**  
  Объяснение, почему потокобезопасность важна при чтении/записи, обзор основных методов `open`, `read`, `write` в многопоточном контексте.

- **Блокировки и синхронизация потоков**  
  Демонстрация использования `threading.Lock`, `RLock`, `Semaphore` для защиты критических секций, обсуждение deadlock и race condition.

- **Практическое упражнение: ThreadPoolExecutor для записи/чтения**  
  Пошаговое решение задачи: параллельное чтение нескольких файлов и запись результатов в общий файл, применение `with ThreadPoolExecutor`.

- **Подведение итогов**  
  Обсуждение ключевых выводов, ответы на вопросы, указание на ресурсы для самостоятельного изучения.
