# Методы и конструкторы

## Цели и задачи лекции
- Понять роль методов и конструкторов в объектно‑ориентированном программировании на Python.  
- Изучить синтаксис и семантику объявления методов и конструкторов.  
- Научиться использовать `self`, `__init__`, `@classmethod`, `@staticmethod`.  
- Практиковать создание и вызов методов, а также наследование и переопределение.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| Класс | Шаблон, определяющий структуру и поведение объектов. |
| Объект | Конкретный экземпляр класса. |
| Метод | Функция, объявленная внутри класса и вызываемая через объект. |
| Конструктор | Специальный метод `__init__`, вызываемый при создании объекта. |
| `self` | Ссылка на текущий объект, передающаяся в методы. |
| `@classmethod` | Декоратор, превращающий метод в класс‑метод. |
| `@staticmethod` | Декоратор, превращающий метод в статический. |
| Наследование | Механизм, позволяющий создавать новый класс на основе существующего. |
| Переопределение | Замена метода родительского класса в подклассе. |
| Полиморфизм | Возможность вызывать одинаковые методы у разных объектов. |

## Основное содержание

### 1. Методы экземпляра (15 мин)
- Объявление метода с первым параметром `self`.  
- Вызов метода через объект.  
- Пример: класс `Person` с методом `greet`.

```python
class Person:
    def __init__(self, name):
        self.name = name

    def greet(self):
        print(f"Привет, меня зовут {self.name}!")
```

### 2. Конструктор `__init__` (15 мин)
- Инициализация атрибутов объекта.  
- Параметры конструктора и их назначение.  
- Пример: класс `Car` с атрибутами `make`, `model`, `year`.

```python
class Car:
    def __init__(self, make, model, year=2020):
        self.make = make
        self.model = model
        self.year = year
```

### 3. Класс‑методы и статические методы (15 мин)
- Отличия: доступ к классу (`cls`) vs. отсутствие доступа.  
- Декораторы `@classmethod` и `@staticmethod`.  
- Пример: фабрика `from_string` и утилита `is_valid_year`.

```python
class Car:
    @classmethod
    def from_string(cls, data):
        make, model, year = data.split('-')
        return cls(make, model, int(year))

    @staticmethod
    def is_valid_year(year):
        return 1886 <= year <= 2100
```

### 4. Наследование и переопределение (15 мин)
- Создание подкласса с помощью `class Subclass(Base):`.  
- Переопределение метода и вызов родительского метода через `super()`.  
- Пример: класс `ElectricCar` наследует `Car`.

```python
class ElectricCar(Car):
    def __init__(self, make, model, year, battery):
        super().__init__(make, model, year)
        self.battery = battery

    def greet(self):
        super().greet()
        print(f"И я электрический автомобиль с батареей {self.battery} kWh.")
```

### 5. Полиморфизм и абстрактные методы (15 мин)
- Определение интерфейса через абстрактный класс (`abc`).  
- Полиморфизм при вызове одинакового метода у разных объектов.  
- Пример: абстрактный класс `Vehicle` и реализация в `Car` и `ElectricCar`.

```python
from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def drive(self):
        pass

class Car(Vehicle):
    def drive(self):
        print("Движение автомобиля.")

class ElectricCar(Vehicle):
    def drive(self):
        print("Движение электрического автомобиля.")
```

## Примеры и иллюстрации

1. **Создание и использование объектов**

```python
p = Person("Алексей")
p.greet()  # Привет, меня зовут Алексей!
```

2. **Фабрика из строки**

```python
c = Car.from_string("Toyota-Corolla-2018")
print(c.make, c.model, c.year)  # Toyota Corolla 2018
```

3. **Полиморфизм**

```python
vehicles = [Car("Honda", "Civic", 2021), ElectricCar("Tesla", "Model 3", 2022, 75)]

for v in vehicles:
    v.drive()
# Движение автомобиля.
# Движение электрического автомобиля.
```

## Выводы по лекции
- Методы и конструкторы являются фундаментальными элементами ООП в Python.  
- `self` обеспечивает доступ к атрибутам объекта, а `__init__` инициализирует его состояние.  
- Класс‑методы и статические методы позволяют работать с классом без создания экземпляра.  
- Наследование и переопределение способствуют расширению функционала и поддержке DRY‑принципа.  
- Полиморфизм обеспечивает гибкость и расширяемость кода.
