# Наследование и переопределение методов

## Цели и задачи лекции
- Понять, как в Python реализуются наследование и переопределение методов.
- Научиться проектировать и использовать иерархии классов.
- Освоить практические примеры применения наследования в реальных задачах.
- Оценить преимущества и ограничения наследования.

## Ключевые понятия
- **Класс** – шаблон для создания объектов с атрибутами и методами.  
- **Наследование** – механизм, позволяющий создать новый класс, наследующий свойства базового.  
- **Базовый (суперкласс)** – класс, от которого наследуются другие.  
- **Потомок (подкласс)** – класс, получающий свойства базового.  
- **Переопределение (overriding)** – замена метода базового класса в подклассе.  
- **super()** – функция, позволяющая вызвать метод базового класса из подкласса.  
- **Многократное наследование** – наследование от более чем одного базового класса.  
- **Миксин** – вспомогательный класс, добавляющий функциональность без полной иерархии.  
- **Полиморфизм** – возможность вызывать один и тот же метод, но он работает по-разному для разных классов.  
- **Метод класса** – метод, объявленный с декоратором `@classmethod`, принимает класс как первый аргумент.

## Основное содержание

### 1. Введение в наследование (15 мин)
- Почему наследование удобно в OOP.  
- Синтаксис объявления наследования:  
  ```python
  class Base:
      def greet(self):
          print("Hello from Base")
  
  class Derived(Base):
      pass
  ```  
- Создание объектов и вызов методов базового класса через потомка.

### 2. Переопределение методов (20 мин)
- Что такое переопределение и как оно работает.  
- Пример:
  ```python
  class Base:
      def describe(self):
          return "Base"
  
  class Derived(Base):
      def describe(self):
          return "Derived"
  ```
- Вызов `super().describe()` внутри переопределённого метода.  
  ```python
  class Derived(Base):
      def describe(self):
          parent = super().describe()
          return f"{parent} with enhancements"
  ```

### 3. Конструкторы и наследование (15 мин)
- Инициализация базового класса через `super().__init__()`.  
  ```python
  class Person:
      def __init__(self, name):
          self.name = name
  
  class Student(Person):
      def __init__(self, name, student_id):
          super().__init__(name)
          self.student_id = student_id
  ```
- Порядок вызова конструкторов при множественном наследовании.

### 4. Многократное наследование и порядок разрешения (15 мин)
- MRO (Method Resolution Order).  
  ```python
  class A: pass
  class B(A): pass
  class C(A): pass
  class D(B, C): pass
  
  print(D.mro())
  ```
- Как решается конфликт одинаковых методов.

### 5. Миксин и композиция (15 мин)
- Что такое миксин и как использовать для добавления функционала.  
  ```python
  class JsonMixin:
      def to_json(self):
          import json
          return json.dumps(self.__dict__)
  
  class User(JsonMixin):
      def __init__(self, name):
          self.name = name
  ```
- Преимущества композиции перед наследованием.

## Примеры и иллюстрации

### 1. Иерархия транспортных средств
```python
class Vehicle:
    def __init__(self, make, model):
        self.make = make
        self.model = model

    def info(self):
        return f"{self.make} {self.model}"

class Car(Vehicle):
    def __init__(self, make, model, doors):
        super().__init__(make, model)
        self.doors = doors

    def info(self):
        base_info = super().info()
        return f"{base_info}, {self.doors} doors"

class Bike(Vehicle):
    def __init__(self, make, model, type):
        super().__init__(make, model)
        self.type = type
```

### 2. Миксин для логирования
```python
class LoggerMixin:
    def log(self, message):
        print(f"[{self.__class__.__name__}] {message}")

class Service(LoggerMixin):
    def run(self):
        self.log("Service started")
```

### 3. Многократное наследование
```python
class Flyer:
    def fly(self):
        print("Flying")

class Swimmer:
    def swim(self):
        print("Swimming")

class Duck(Flyer, Swimmer):
    pass

d = Duck()
d.fly()
d.swim()
```

## Выводы по лекции
- Наследование позволяет повторно использовать код и строить иерархии классов.  
- Переопределение методов даёт гибкость, но требует аккуратного обращения с `super()`.  
- Многократное наследование возможно, но требует понимания MRO.  
- Миксины и композиция часто предпочтительнее, когда нужны только отдельные фичи.
