# Составление введения. Аналитический обзор и разработка проекта приложения

## Цели и задачи лекции
- Понять структуру и назначение введения в проекте.
- Освоить методы проведения аналитического обзора.
- Научиться формулировать требования к проекту приложения.
- Практически применить навыки планирования и документирования.

## Ключевые понятия
- **Введение** – часть документа, описывающая цель, контекст и область применения проекта.
- **Аналитический обзор** – систематический анализ существующих решений и требований.
- **Требования** – перечень функций и характеристик, которые должно удовлетворять приложение.
- **План проекта** – график, ресурсы и этапы реализации.
- **Прототип** – упрощённая версия продукта для проверки концепции.
- **Управление рисками** – выявление и оценка потенциальных проблем.
- **Методология Agile** – гибкий подход к управлению проектом.
- **Use‑case** – описание взаимодействия пользователя с системой.
- **Wireframe** – схематическое представление интерфейса.
- **Критерии приемки** – условия, при которых продукт считается завершённым.

## Основное содержание

### 1. Введение в проект (15 мин)
- Что такое введение и зачем оно нужно.
- Элементы введения: цель, задачи, аудитория, ограничения.
- Пример шаблона введения.

```markdown
# Введение
## Цель проекта
Разработать приложение для учета задач студентов.
## Задачи
1. Создать базу данных.
2. Реализовать интерфейс пользователя.
3. Обеспечить безопасность данных.
## Аудитория
Студенты и преподаватели колледжа.
```

### 2. Аналитический обзор (20 мин)
- Сбор и анализ требований.
- Методы исследования: интервью, опросы, анализ конкурентов.
- Составление матрицы требований.

```python
# Пример кода для сбора отзывов через форму
def collect_feedback(user_id, comment):
    with open('feedback.txt', 'a') as f:
        f.write(f"{user_id}: {comment}\n")
```

### 3. Формулирование требований (20 мин)
- Разделение требований на функциональные и нефункциональные.
- Использование шаблонов «User Story» и «Acceptance Criteria».
- Приоритизация требований (MoSCoW).

```text
User Story: Как студент, я хочу добавлять задачи, чтобы отслеживать дедлайны.
Acceptance Criteria:
- Пользователь может создать задачу через форму.
- Задача сохраняется в базе данных.
```

### 4. План проекта и управление рисками (15 мин)
- Создание дорожной карты (roadmap).
- Распределение ролей и ресурсов.
- Идентификация рисков и их оценка.

```markdown
# План проекта
1. Анализ требований – 01.09–07.09
2. Проектирование – 08.09–14.09
3. Реализация – 15.09–30.09
4. Тестирование – 01.10–07.10
```

### 5. Прототипирование и Wireframe (10 мин)
- Инструменты: Figma, Balsamiq, Sketch.
- Создание простых макетов интерфейса.

```markdown
# Wireframe
- Главная страница
  - Список задач
  - Кнопка «Добавить задачу»
- Страница добавления
  - Поля: Название, Описание, Дедлайн
```

## Примеры и иллюстрации
- Показан код простого CRUD‑приложения на Flask.
- Схема архитектуры клиент‑сервер.
- График Gantt‑диаграммы проекта.

```python
from flask import Flask, request, jsonify

app = Flask(__name__)

tasks = []

@app.route('/tasks', methods=['GET'])
def get_tasks():
    return jsonify(tasks)

@app.route('/tasks', methods=['POST'])
def add_task():
    data = request.json
    tasks.append(data)
    return jsonify(data), 201

if __name__ == '__main__':
    app.run(debug=True)
```
