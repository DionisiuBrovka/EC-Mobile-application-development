# Виджеты и жесты

## Цели и задачи лекции
- Понять, что такое виджеты и жесты в пользовательских интерфейсах.
- Изучить основные типы виджетов и жестов, их назначение и применение.
- Научиться реализовывать простые виджеты и жесты на Python с использованием библиотеки Tkinter.
- Оценить влияние корректного использования виджетов и жестов на удобство и эффективность программ.

## Ключевые понятия
- **Виджет** – элемент графического интерфейса, предоставляющий пользователю способ взаимодействия с программой (кнопка, поле ввода, список и т.д.).
- **Жест** – последовательность движений мыши или касаний, распознаваемая системой как команда (смахивание, двойной клик, масштабирование).
- **Событие** – реакция на действие пользователя (нажатие кнопки, перемещение мыши и пр.).
- **Контейнер** – виджет, в котором размещаются другие виджеты (Frame, Toplevel).
- **Callback** – функция‑обработчик, вызываемая при возникновении события.
- **Пакет** – группа виджетов, объединённых общей функциональностью (например, меню).
- **Графический интерфейс пользователя (GUI)** – совокупность виджетов и событий, создающих взаимодействие с пользователем.
- **Кросс‑платформенность** – способность программы работать на разных ОС без изменения кода.
- **Адаптивность** – возможность интерфейса автоматически менять размер и расположение виджетов под экран пользователя.
- **Accessibility (доступность)** – обеспечение интерфейса для людей с ограниченными возможностями.

## Основное содержание

### 1. Введение в виджеты (20 мин)
- Что такое виджеты и зачем они нужны.
- Классификация виджетов: базовые, комбинированные, интерактивные.
- Пример простого окна с кнопкой.

```python
import tkinter as tk

root = tk.Tk()
root.title("Пример виджета")

button = tk.Button(root, text="Нажми меня")
button.pack(pady=10)

root.mainloop()
```

### 2. Работа с событиями и callback‑функциями (25 мин)
- Определение событий (<<Button-1>>, <<Enter>>, <<Leave>> и др.).
- Привязка событий к виджетам (`widget.bind()`).
- Параметры событий (event.x, event.y).

```python
def on_click(event):
    print(f"Клик в координатах {event.x}, {event.y}")

button.bind("<Button-1>", on_click)
```

### 3. Расширенные виджеты и их свойства (20 мин)
- Поле ввода (`Entry`), текстовое поле (`Text`), список (`Listbox`), выпадающий список (`OptionMenu`).
- Настройка внешнего вида: цвет, размер шрифта, рамки.
- Пример формы с несколькими виджетами.

```python
label = tk.Label(root, text="Имя:")
label.pack()
entry = tk.Entry(root)
entry.pack()
```

### 4. Жесты мыши и их обработка (15 мин)
- Смахивание, двойной клик, выделение.
- Использование `event.widget` для определения виджета.
- Минимальный пример с распознаванием двойного клика.

```python
def on_double_click(event):
    print("Двойной клик!")

button.bind("<Double-Button-1>", on_double_click)
```

### 5. Создание собственного виджета (10 мин)
- Наследование от `tk.Frame`.
- Инкапсуляция логики и внешнего вида.
- Пример простого слайдера.

```python
class Slider(tk.Frame):
    def __init__(self, master, **kwargs):
        super().__init__(master, **kwargs)
        self.scale = tk.Scale(self, from_=0, to=100, orient='horizontal')
        self.scale.pack()
```

## Примеры и иллюстрации
- Схема взаимодействия виджетов и событий:

```
[Кнопка] --(нажатие)--> [Обработчик] --> [Изменение текста]
```

- Код реализации комбинированного виджета «Поле + Кнопка»:

```python
class SearchBar(tk.Frame):
    def __init__(self, master, callback, **kwargs):
        super().__init__(master, **kwargs)
        self.entry = tk.Entry(self)
        self.entry.pack(side='left', fill='x', expand=True)
        self.btn = tk.Button(self, text="Найти", command=callback)
        self.btn.pack(side='right')
```

## Выводы по лекции
- Виджеты и жесты – фундаментальные элементы GUI, обеспечивающие взаимодействие пользователя с программой.
- Правильная организация событий и callback‑функций повышает отзывчивость интерфейса.
- Расширенные виджеты и собственные компоненты позволяют создавать гибкие и удобные приложения.
- Понимание принципов работы жестов улучшает UX и делает программы более интуитивными.
