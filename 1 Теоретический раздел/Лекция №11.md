# Многопоточное программирование

## Цели и задачи лекции  
1. Понять, что такое многопоточное программирование и зачем оно нужно.  
2. Изучить основные модели потоков и синхронизации в Python.  
3. Научиться создавать и управлять потоками, а также избегать гонок и взаимоблокировок.  
4. Реализовать простые практические примеры с использованием модуля `threading`.

## Ключевые понятия  
- **Поток (Thread)** – легковесный процесс, исполняемый внутри процесса.  
- **Многопоточность (Multithreading)** – одновременное выполнение нескольких потоков.  
- **Блокировка (Lock)** – механизм синхронизации, запрещающий одновременный доступ к ресурсу.  
- **Взаимоблокировка (Deadlock)** – ситуация, когда два и более потока ждут друг друга.  
- **Критическая секция (Critical section)** – участок кода, доступ к которому должен быть эксклюзивным.  
- **Пул потоков (ThreadPool)** – коллекция рабочих потоков, которые обслуживают задачи из очереди.  
- **Событие (Event)** – объект синхронизации, сигнализирующий о наступлении события.  
- **Семафор (Semaphore)** – ограничивает количество потоков, одновременно использующих ресурс.  
- **Флаг завершения (Flag)** – переменная, сигнализирующая о необходимости остановки потока.  
- **GIL (Global Interpreter Lock)** – глобальная блокировка интерпретатора CPython, ограничивающая параллелизм.

## Основное содержание  

### 1. Введение в многопоточность (10 мин)  
- Почему многопоточность полезна: параллелизм, реактивность, экономия ресурсов.  
- Ограничения: GIL в CPython, накладные расходы на синхронизацию.  

### 2. Создание потоков в Python (15 мин)  
- Класс `threading.Thread`: конструктор, методы `start()`, `join()`.  
- Пример простого потока.  
```python
import threading

def worker(name):
    print(f"Поток {name} запущен")

t = threading.Thread(target=worker, args=("A",))
t.start()
t.join()
```  
- Разница между `Thread` и `Process` (модуль `multiprocessing`).  

### 3. Синхронизация потоков (20 мин)  
- Критическая секция и блокировки (`Lock`).  
```python
lock = threading.Lock()
lock.acquire()
# критический код
lock.release()
```  
- Контекстный менеджер `with lock:`.  
- Семафоры (`Semaphore`) и их применение.  
- События (`Event`) для координации потоков.  
- Пример гонки данных и как её избежать.  

### 4. Пул потоков (15 мин)  
- Класс `ThreadPoolExecutor` из `concurrent.futures`.  
- Параметры `max_workers`, `map`, `submit`.  
```python
from concurrent.futures import ThreadPoolExecutor

def calc(x):
    return x * x

with ThreadPoolExecutor(max_workers=4) as pool:
    results = pool.map(calc, range(10))
    print(list(results))
```  
- Плюсы и минусы использования пула.  

### 5. Избежание взаимоблокировок (10 мин)  
- Причины взаимоблокировки.  
- Стратегии: порядок захвата блокировок, таймауты, `threading.RLock`.  
- Пример простого deadlock и его решение.  

### 6. Практическое задание (10 мин)  
- Создать программу, которая одновременно скачивает несколько файлов из интернета, используя потоки, и выводит прогресс.

## Примеры и иллюстрации  

### Параллельные вычисления  
```python
import threading

def compute(nums, result, idx, lock):
    total = sum(nums)
    with lock:
        result[idx] = total

numbers = [list(range(1000000)) for _ in range(4)]
results = [0] * 4
lock = threading.Lock()
threads = []

for i, nums in enumerate(numbers):
    t = threading.Thread(target=compute, args=(nums, results, i, lock))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print(results)
```

### Пул потоков для обработки файлов  
```python
import concurrent.futures
import time

def download_file(url):
    time.sleep(1)  # имитация скачивания
    return f"Файл {url} скачан"

urls = [f"http://example.com/file{i}.txt" for i in range(5)]

with concurrent.futures.ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(download_file, url) for url in urls]
    for future in concurrent.futures.as_completed(futures):
        print(future.result())
```

## Выводы по лекции  
- Многопоточность позволяет эффективно использовать ресурсы, но требует аккуратной синхронизации.  
- В CPython GIL ограничивает параллелизм CPU‑интенсивных задач, но многопоточность всё же полезна для I/O‑операций.  
- Правильное применение `Lock`, `Semaphore`, `Event` и `ThreadPoolExecutor` снижает риск гонок и взаимоблокировок.
