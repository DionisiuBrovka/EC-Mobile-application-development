# Анализ ОКР №1. Компоненты Android-приложения

## Цели и задачи лекции
- Познакомить студентов с архитектурой Android‑приложения.
- Выделить основные компоненты и их взаимодействие.
- Показать, как реализовать простое приложение, используя эти компоненты.
- Объяснить роль Manifest и ресурсов в проекте Android.

## Ключевые понятия
- **Activity** – экран пользовательского интерфейса, управляемый жизненным циклом.
- **Fragment** – модуль пользовательского интерфейса, переиспользуемый в Activity.
- **Service** – фоновый компонент, выполняющий долгие операции.
- **BroadcastReceiver** – компонент, реагирующий на системные и пользовательские события.
- **ContentProvider** – механизм обмена данными между приложениями.
- **Intent** – объект, передающий информацию и инициирующий действия.
- **Manifest** – метаданные приложения, включая компоненты и разрешения.
- **Resources** – набор файлов (строк, изображений, макетов), доступных в коде.
- **Layout** – XML‑файл, описывающий пользовательский интерфейс.
- **View** – базовый элемент пользовательского интерфейса.

## Основное содержание

### 1. Структура проекта Android (15 мин)
- Папка `app/` → `src/main/` → `java/`, `res/`, `AndroidManifest.xml`.
- Разделение кода и ресурсов: Java/Kotlin и XML.

#### 1.1. AndroidManifest.xml (5 мин)
```xml
<manifest package="com.example.myapp">
    <application
        android:label="@string/app_name"
        android:theme="@style/AppTheme">
        <activity android:name=".MainActivity" />
    </application>
</manifest>
```
- Определяет пакеты, разрешения, компоненты.

#### 1.2. Ресурсы (10 мин)
- Строки (`strings.xml`), изображения (`drawable/`), стили (`styles.xml`), макеты (`layout/`).
- Доступ через `R` (auto‑генерируемый класс).

### 2. Activity и её жизненный цикл (20 мин)
- Методы `onCreate()`, `onStart()`, `onResume()`, `onPause()`, `onStop()`, `onDestroy()`.
- Пример простого Activity:
```kotlin
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
    }
}
```
- Объяснение переходов между состояниями.

### 3. Fragment – переиспользуемый UI (15 мин)
- Создание Fragment‑класса и его привязка к Activity.
```kotlin
class SampleFragment : Fragment() {
    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View? {
        return inflater.inflate(R.layout.fragment_sample, container, false)
    }
}
```
- Вставка фрагмента в Activity через XML или динамически.

### 4. Фоновая работа: Service (10 мин)
- Виды Service: `started`, `bound`, `foreground`.
- Пример простого `IntentService`:
```kotlin
class DownloadService : IntentService("DownloadService") {
    override fun onHandleIntent(intent: Intent?) {
        // Долговремая операция
    }
}
```

### 5. Обработка событий: BroadcastReceiver (5 мин)
- Регистрация в Manifest и динамическая регистрация.
```kotlin
class BootReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        // Реакция на событие
    }
}
```

### 6. Обмен данными: ContentProvider (5 мин)
- Создание `SQLiteOpenHelper`, реализация CRUD‑операций.
- Пример вызова из другого приложения через `content://`.

## Выводы по лекции
- Android‑приложение состоит из набора компонентов, каждый из которых отвечает за определённую часть функционала.
- `Activity` и `Fragment` управляют пользовательским интерфейсом, `Service` – фоновыми задачами, `BroadcastReceiver` – событиями, `ContentProvider` – совместным доступом к данным.
- Понимание Manifest, ресурсов и жизненного цикла критически важно для корректной работы приложения.
