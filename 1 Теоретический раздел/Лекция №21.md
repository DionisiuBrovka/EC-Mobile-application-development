# Курсоры. Получение данных и выполнение в фоновом режиме

## Цели и задачи лекции  
1. Понять роль курсоров в работе с базами данных и в Python.  
2. Научиться получать данные из внешних источников (CSV, API, базы).  
3. Освоить методы асинхронного и фонового выполнения задач.  
4. Практически применить полученные знания в небольших проектах.

## Ключевые понятия  
- **Курсор** – указатель, определяющий текущую строку в наборе результатов запроса.  
- **SELECT** – оператор SQL, возвращающий набор строк.  
- **fetchone() / fetchall()** – методы курсора для получения одной или всех строк.  
- **asyncio** – стандартная библиотека Python для асинхронного программирования.  
- **ThreadPoolExecutor** – класс из `concurrent.futures` для выполнения задач в потоках.  
- **ProcessPoolExecutor** – аналог для процессов.  
- **Event loop** – цикл событий, управляющий асинхронными задачами.  
- **Callback** – функция, вызываемая по завершении асинхронной операции.  
- **Future** – объект, представляющий результат, который появится позже.  
- **Batch processing** – пакетная обработка данных.

## Основное содержание  

### 1. Работа с курсорами (30 минут)  
#### 1.1. Создание и использование курсора  
```python
import sqlite3

conn = sqlite3.connect('example.db')
cursor = conn.cursor()
cursor.execute('SELECT * FROM users')
row = cursor.fetchone()
while row:
    print(row)
    row = cursor.fetchone()
```
#### 1.2. fetchone() vs fetchall()  
- `fetchone()` – по одной строке, экономит память.  
- `fetchall()` – сразу все строки, удобно при небольших наборах.  
#### 1.3. Переход по курсору  
- `cursor.scroll(10, 'relative')` – смещение.  
- `cursor.scroll(0, 'absolute')` – к первой строке.  

### 2. Получение данных из внешних источников (25 минут)  
#### 2.1. Чтение CSV  
```python
import csv
with open('data.csv', newline='') as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row['name'])
```
#### 2.2. Запросы к REST‑API  
```python
import requests
response = requests.get('https://api.example.com/users')
data = response.json()
for user in data:
    print(user['id'])
```
#### 2.3. Подключение к удалённой БД (PostgreSQL)  
```python
import psycopg2
conn = psycopg2.connect(
    dbname='db', user='user', password='pass', host='localhost'
)
cursor = conn.cursor()
cursor.execute('SELECT id, name FROM customers')
for row in cursor:
    print(row)
```

### 3. Фоновое и асинхронное выполнение (35 минут)  
#### 3.1. Асинхронные функции с asyncio  
```python
import asyncio
async def fetch_data(url):
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as resp:
            return await resp.json()

async def main():
    data = await fetch_data('https://api.example.com/items')
    print(data)

asyncio.run(main())
```
#### 3.2. Параллельные задачи с ThreadPoolExecutor  
```python
from concurrent.futures import ThreadPoolExecutor, as_completed

def heavy_task(i):
    return i * i

with ThreadPoolExecutor(max_workers=5) as executor:
    futures = [executor.submit(heavy_task, i) for i in range(10)]
    for future in as_completed(futures):
        print(future.result())
```
#### 3.3. Параллельные задачи с ProcessPoolExecutor  
```python
from concurrent.futures import ProcessPoolExecutor

def compute(i):
    return i ** 2

with ProcessPoolExecutor() as executor:
    results = executor.map(compute, range(10))
    print(list(results))
```
#### 3.4. Синхронизация и обратные вызовы  
```python
def callback(future):
    print('Result:', future.result())

future = executor.submit(heavy_task, 5)
future.add_done_callback(callback)
```

## Выводы по лекции  
- Курсоры позволяют эффективно обрабатывать результаты запросов, управляя памятью.  
- Данные можно получать из различных источников (CSV, API, БД) с помощью стандартных библиотек.  
- Асинхронные и фоновые техники повышают производительность и отзывчивость программ.  
- Выбор подхода зависит от характера задачи: I/O‑зависимые операции – `asyncio`; CPU‑зависимые – `ProcessPoolExecutor`.
