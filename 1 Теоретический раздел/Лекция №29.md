# Исправление ошибок. Создание диаграммы классов

## Цели и задачи лекции
- Понять основные типы ошибок в Python и способы их выявления.
- Научиться использовать инструменты отладки и тестирования.
- Освоить принцип создания и интерпретации диаграмм классов.
- Применить полученные знания на практике при разработке небольших проектов.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **Исключение (Exception)** | Событие, прерывающее нормальный поток выполнения программы. |
| **Трассировка стека (Stack trace)** | Список вызовов функций, ведущий к месту возникновения исключения. |
| **try/except** | Конструкция, позволяющая перехватывать и обрабатывать исключения. |
| **finally** | Блок, выполняемый независимо от того, возникло ли исключение. |
| **PEP 8** | Стандарт стиля кода для Python. |
| **Диаграмма классов (Class diagram)** | UML-диаграмма, показывающая классы, их атрибуты, методы и связи. |
| **Атрибут** | Переменная, принадлежащая классу. |
| **Метод** | Функция, определённая внутри класса. |
| **Наследование** | Механизм, позволяющий создавать новый класс на основе существующего. |
| **Инкапсуляция** | Сокрытие внутренней реализации класса от внешнего мира. |

## Основное содержание

### 1. Ошибки и исключения (15 мин)
- Различие между ошибками времени компиляции и времени выполнения.
- Частые типы ошибок: `SyntaxError`, `NameError`, `TypeError`, `IndexError` и др.
- Пример возникновения ошибки:

```python
def divide(a, b):
    return a / b

print(divide(10, 0))  # ZeroDivisionError
```

### 2. Обработка исключений (20 мин)
- Синтаксис `try/except`:

```python
try:
    result = divide(10, 0)
except ZeroDivisionError as e:
    print(f"Ошибка: {e}")
```

- Блок `else` и `finally`.
- Логирование ошибок с помощью модуля `logging`.

### 3. Инструменты отладки (15 мин)
- Использование `pdb`:

```python
import pdb; pdb.set_trace()
```

- Параметры `breakpoint()`, `step`, `next`, `continue`.
- Интеграция с IDE (PyCharm, VSCode).

### 4. Тестирование (15 мин)
- Понятие unit-тестов.
- Модуль `unittest`:

```python
import unittest

class TestDivide(unittest.TestCase):
    def test_divide(self):
        self.assertEqual(divide(10, 2), 5)

if __name__ == "__main__":
    unittest.main()
```

- Тесты с использованием `assertRaises`.

### 5. Диаграмма классов (15 мин)
- Основные элементы диаграммы: класс, атрибуты, методы, ассоциации, наследование.
- Пример UML-диаграммы в виде текста:

```
+-----------------+        +-----------------+
|   Vehicle       |        |   Car           |
|-----------------|        |-----------------|
| - make: str     |<>------| - model: str    |
| - year: int     |        | - seats: int    |
| + start(): void |        | + honk(): void  |
+-----------------+        +-----------------+
```

- Создание диаграммы в Python с помощью `pydot` и `graphviz`:

```python
from graphviz import Digraph

dot = Digraph()
dot.node('Vehicle')
dot.node('Car')
dot.edge('Vehicle', 'Car', constraint='false')
print(dot.source)
```

## Примеры и иллюстрации
```python
class Person:
    def __init__(self, name, age):
        self.name = name          # атрибут
        self.age = age

    def greet(self):               # метод
        print(f"Привет, меня зовут {self.name}")

class Employee(Person):            # наследование
    def __init__(self, name, age, employee_id):
        super().__init__(name, age)
        self.employee_id = employee_id

    def work(self):
        print(f"{self.name} работает над задачами.")
```

## Выводы по лекции
- Исключения позволяют безопасно обрабатывать ошибки, не прерывая программу.

- Диаграммы классов помогают визуализировать структуру и связи в объектно‑ориентированных системах.
