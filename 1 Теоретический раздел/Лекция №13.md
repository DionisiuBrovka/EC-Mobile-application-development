# Многопоточное программирование при работе с файлами

## Цели и задачи лекции
1. Понять, зачем нужны многопоточные операции при работе с файлами.  
2. Изучить основные инструменты многопоточности в Python (модуль `threading`).  
3. Научиться безопасно читать и писать файлы из нескольких потоков.  
4. Практически реализовать простое многопоточное приложение, обрабатывающее файлы.

## Ключевые понятия
- **Многопоточность** – одновременное выполнение нескольких потоков в одном процессе.  
- **Поток (Thread)** – минимальная единица выполнения, часть процесса.  
- **Блокировка (Lock)** – механизм синхронизации, предотвращающий одновременный доступ к ресурсу.  
- **Критическая секция** – участок кода, который должен выполняться эксклюзивно одним потоком.  
- **Файл‑объект** – объект, представляющий открытый файл, с которым выполняются операции чтения/записи.  
- **Потокобезопасность** – свойство программы, при котором параллельные операции не приводят к некорректным результатам.  
- **Пул потоков (ThreadPool)** – коллекция готовых потоков, из которой берутся рабочие потоки для задач.  
- **Кодек** – набор символов для кодирования/декодирования текстовых файлов.  
- **Потоковый режим** – режим открытия файла (`'r'`, `'w'`, `'a'`, `'rb'` и т.п.).  
- **Синхронный ввод‑вывод** – операции ввода‑вывода, которые блокируют поток до завершения.

## Основное содержание

### 1. Введение в многопоточность (5 мин)
- Почему многопоточность полезна при работе с файлами.  
- Ограничения Python: GIL и его влияние на CPU‑intensive задачи.  
- Когда многопоточность помогает: I/O‑bound операции, чтение/запись больших файлов.

### 2. Модуль `threading` и базовые понятия (10 мин)
- Создание потоков (`threading.Thread`).  
- Запуск и ожидание (`start()`, `join()`).  
- Атрибуты потока: `name`, `daemon`.  
- Пример простого потока, который печатает сообщение.

```python
import threading
import time

def worker(name):
    for i in range(3):
        print(f"{name} - шаг {i+1}")
        time.sleep(0.5)

t = threading.Thread(target=worker, args=("Поток-1",))
t.start()
t.join()
```

### 3. Синхронизация доступа к файлу (10 мин)
- Проблема одновременного доступа к одному файлу.  
- Использование `threading.Lock` для защиты критической секции.  
- Пример чтения из файла с блокировкой.

```python
lock = threading.Lock()
file_path = 'data.txt'

def read_file():
    with lock:
        with open(file_path, 'r', encoding='utf-8') as f:
            data = f.read()
            print(f"Прочитано: {len(data)} символов")

t1 = threading.Thread(target=read_file, name='Читатель-1')
t2 = threading.Thread(target=read_file, name='Читатель-2')
t1.start(); t2.start()
t1.join(); t2.join()
```

### 4. Параллельное чтение больших файлов (10 мин)
- Разделение файла на блоки по строкам/байтам.  
- Создание потоков, каждый читает свой блок.  
- Сбор результатов в главный поток.

```python
def read_chunk(start, size, result, idx):
    with lock:
        with open(file_path, 'r', encoding='utf-8') as f:
            f.seek(start)
            chunk = f.read(size)
            result[idx] = chunk

chunks = 4
size = 1024 * 1024  # 1 МБ
results = [None] * chunks
threads = []

for i in range(chunks):
    t = threading.Thread(target=read_chunk, args=(i*size, size, results, i))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

full_data = ''.join(results)
print(f"Объём прочитанного: {len(full_data)} байт")
```

### 5. Параллельная запись в файл (5 мин)
- Как безопасно писать в один файл из разных потоков.  
- Использование `lock` и режима `a` (append).  
- Минимизация блокировок: буферизация данных в памяти.

```python
def write_chunk(data, lock, path='out.txt'):
    with lock:
        with open(path, 'a', encoding='utf-8') as f:
            f.write(data)

threads = []
for i in range(4):
    t = threading.Thread(target=write_chunk, args=(f"Блок {i}\n", lock))
    threads.append(t)
    t.start()

for t in threads:
    t.join()
```

### 6. Пул потоков и `concurrent.futures` (5 мин)
- Почему использовать пул потоков.  
- Пример `ThreadPoolExecutor` для обработки файлов.  
- Преимущества: управление количеством потоков, упрощённое управление.

```python
from concurrent.futures import ThreadPoolExecutor

def process_file(path):
    with open(path, 'r', encoding='utf-8') as f:
        return len(f.readlines())

paths = ['file1.txt', 'file2.txt', 'file3.txt', 'file4.txt']
with ThreadPoolExecutor(max_workers=4) as executor:
    results = executor.map(process_file, paths)

for path, lines in zip(paths, results):
    print(f"{path}: {lines} строк")
```

## Примеры и иллюстрации
- Пояснительные блоки кода приведены выше.  
- Схема потоков (текстовая) для параллельного чтения:

```
[Главный поток] → [Поток-1] читаёт блок 1
                 → [Поток-2] читаёт блок 2
                 → [Поток-3] читаёт блок 3
                 → [Поток-4] читаёт блок 4
[Главный поток] собирает блоки → формирует полный файл
```

## Выводы по лекции
- Многопоточная работа с файлами позволяет существенно ускорить I/O‑bound задачи.  
- Для безопасного доступа к файлам критически важна синхронизация (`Lock`).  
- Пул потоков упрощает управление количеством активных потоков и повышает читаемость кода.  
- В Python GIL не препятствует параллельному вводу‑выводу, но важно избегать избыточного переключения потоков.
