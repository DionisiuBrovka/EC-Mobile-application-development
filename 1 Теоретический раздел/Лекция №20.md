# Работа с базой данных SQLite

## Цели и задачи лекции
1. Ознакомить студентов с концепцией реляционных баз данных и особенностями SQLite.  
2. Научить создавать, модифицировать и удалять таблицы и записи.  
3. Познакомить с основными SQL‑операциями: SELECT, INSERT, UPDATE, DELETE, JOIN.  
4. Показать, как интегрировать SQLite в программы на Python через модуль `sqlite3`.

## Ключевые понятия
- **Реляционная база данных** – структура данных, основанная на таблицах с взаимосвязанными строками.  
- **SQL (Structured Query Language)** – язык запросов для работы с реляционными БД.  
- **Таблица (table)** – набор строк и столбцов, где каждая строка представляет запись.  
- **Ключ (key)** – поле, уникально идентифицирующее строку (PRIMARY KEY).  
- **Индексы (indexes)** – структуры, ускоряющие поиск по столбцам.  
- **Курсор (cursor)** – объект, позволяющий выполнять запросы и получать результаты.  
- **Транзакция (transaction)** – атомарный набор операций, завершающийся `COMMIT` или `ROLLBACK`.  
- **Встроенный сервер (embedded server)** – БД, работающая внутри приложения (как SQLite).  
- **ACID** – набор свойств транзакций: Atomicity, Consistency, Isolation, Durability.  
- **ORM (Object‑Relational Mapping)** – абстракция, позволяющая работать с БД через объекты.

## Основное содержание

### 1. Введение в SQLite (15 мин)
- История и назначение SQLite.  
- Преимущества: отсутствие сервера, простота установки, переносимость.  
- Установка и проверка версии через команду `sqlite3 --version`.

### 2. Создание и открытие базы данных (10 мин)
- Создание файла БД: `sqlite3 mydb.sqlite`.  
- Открытие БД в Python:  
  ```python
  import sqlite3
  conn = sqlite3.connect('mydb.sqlite')
  cursor = conn.cursor()
  ```

### 3. Проектирование таблиц (20 мин)
- Определение схемы: `CREATE TABLE`.  
- Типы данных SQLite: INTEGER, TEXT, BLOB, REAL, NULL.  
- Ограничения: PRIMARY KEY, NOT NULL, UNIQUE, CHECK, FOREIGN KEY.  
- Пример создания таблицы `students`.  
  ```sql
  CREATE TABLE students (
      id INTEGER PRIMARY KEY,
      name TEXT NOT NULL,
      age INTEGER,
      email TEXT UNIQUE
  );
  ```

### 4. Работа с данными (25 мин)
- **INSERT**: добавление записей.  
  ```sql
  INSERT INTO students (name, age, email) VALUES ('Иван', 20, 'ivan@example.com');
  ```  
- **SELECT**: выборка данных, условия `WHERE`, сортировка `ORDER BY`, ограничение `LIMIT`.  
  ```sql
  SELECT * FROM students WHERE age > 18 ORDER BY name ASC LIMIT 10;
  ```  
- **UPDATE**: изменение записей.  
  ```sql
  UPDATE students SET age = age + 1 WHERE id = 1;
  ```  
- **DELETE**: удаление записей.  
  ```sql
  DELETE FROM students WHERE id = 1;
  ```

### 5. Транзакции и управление изменениями (10 мин)
- Автоматический режим автокоммита.  
- Ручной контроль: `BEGIN`, `COMMIT`, `ROLLBACK`.  
- Пример отката при ошибке.

### 6. Связи между таблицами (10 мин)
- FOREIGN KEY и ограничение целостности.  
- Пример двух таблиц: `students` и `grades`.  
  ```sql
  CREATE TABLE grades (
      id INTEGER PRIMARY KEY,
      student_id INTEGER,
      subject TEXT,
      score REAL,
      FOREIGN KEY (student_id) REFERENCES students(id)
  );
  ```  
- JOIN‑операции: `INNER JOIN`, `LEFT JOIN`.

### 7. Интеграция с Python (10 мин)
- Выполнение запросов через `cursor.execute`.  
- Получение результатов: `fetchone`, `fetchall`.  
- Параметризованные запросы для предотвращения SQL‑инъекций.  
  ```python
  cursor.execute("SELECT * FROM students WHERE id = ?", (student_id,))
  ```

### 8. Практическое задание (15 мин)
- Создать БД `library.sqlite`.  
- Таблицы: `authors`, `books`, `borrowers`.  
- Написать скрипт, добавляющий, обновляющий и выводящий данные.

## Выводы по лекции
- SQLite – удобный инструмент для небольших и средних приложений.  
- Основные SQL‑операции позволяют полноценно управлять данными.  
- Транзакции обеспечивают целостность и надёжность.  
- Интеграция с Python проста и безопасна при использовании параметризованных запросов.
