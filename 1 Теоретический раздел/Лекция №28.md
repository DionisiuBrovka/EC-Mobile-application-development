## Цели и задачи лекции

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| Тест | Проверка работы программы с целью обнаружения дефектов. |
| Тестовый случай | Конкретный набор входных данных и ожидаемый результат. |
| Тестовый набор | Коллекция тестовых случаев, покрывающих определённый аспект программы. |
| Юнит‑тест | Тест, проверяющий работу небольшого, изолированного блока кода. |
| Интеграционное тестирование | Тестирование взаимодействия между модулями. |
| Системное тестирование | Проверка целостности всей системы. |
| Регрессионное тестирование | Повторное выполнение тестов после изменений для выявления новых дефектов. |
| Фреймворк тестирования | Инструмент, облегчающий написание и запуск тестов. |
| Mock | Заглушка, имитирующая поведение внешних зависимостей. |
| CI/CD | Автоматизированный процесс сборки, тестирования и развёртывания. |

## Примеры и иллюстрации
```python
# Пример юнит‑теста с unittest
import unittest

def add(a, b):
    return a + b

class TestAdd(unittest.TestCase):
    def test_positive_numbers(self):
        self.assertEqual(add(2, 3), 5)

    def test_negative_numbers(self):
        self.assertEqual(add(-1, -1), -2)

if __name__ == "__main__":
    unittest.main()
```

```python
# Пример теста с pytest и параметризацией
import pytest

def multiply(a, b):
    return a * b

@pytest.mark.parametrize("a,b,expected", [
    (2, 3, 6),
    (0, 5, 0),
    (-1, 4, -4)
])
def test_multiply(a, b, expected):
    assert multiply(a, b) == expected
```

```python
# Использование mock для тестирования внешнего API
from unittest.mock import patch
import requests

def get_user_name(user_id):
    response = requests.get(f"https://api.example.com/users/{user_id}")
    return response.json()["name"]

def test_get_user_name():
    with patch('requests.get') as mock_get:
        mock_get.return_value.json.return_value = {"name": "Alice"}
        assert get_user_name(42) == "Alice"
```
