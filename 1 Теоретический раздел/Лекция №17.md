# Фрагменты. Жизненный цикл фрагментов

## Цели и задачи лекции
1. Понять, что такое фрагменты и как они применяются в программировании.  
2. Изучить этапы жизненного цикла фрагмента.  
3. Научиться создавать, инициализировать и управлять фрагментами через практические примеры.  
4. Оценить преимущества и ограничения использования фрагментов.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **Фрагмент** | Независимый модуль пользовательского интерфейса, который может быть вложен в активность. |
| **Контейнер** | Компонент (обычно `Activity`), в котором размещается фрагмент. |
| **Вью** (`View`) | Графический элемент, отображаемый фрагментом. |
| **Пакет** (`Bundle`) | Словарь, передаваемый фрагменту для инициализации. |
| **Фрагментный менеджер** (`FragmentManager`) | Инструмент управления жизненным циклом фрагментов. |
| **Фрагментный транзакция** (`FragmentTransaction`) | Операция над фрагментами (добавление, удаление, замена). |
| **Состояние** | Текущее состояние фрагмента в рамках его жизненного цикла. |
| **Кэш** | Хранение в памяти ранее созданных фрагментов для быстрого доступа. |
| **Обратный стек** (`Back Stack`) | Хранит историю транзакций для навигации назад. |
| **Переход** (`Transition`) | Анимация при изменении фрагментов. |

## Основное содержание

### 1. Введение в фрагменты (10 мин)
- История появления фрагментов в Android.  
- Зачем нужны фрагменты: переиспользуемость, модульность, гибкость UI.  
- Сравнение с обычными окнами и виджетами.

### 2. Структура фрагмента (15 мин)
- Класс `Fragment` и его наследники.  
- Переопределяемые методы: `onAttach`, `onCreate`, `onCreateView`, `onActivityCreated`, `onStart`, `onResume`, `onPause`, `onStop`, `onDestroyView`, `onDestroy`, `onDetach`.  
- Пример кода: базовый фрагмент с простым UI.

```java
public class SimpleFragment extends Fragment {
    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container,
                             Bundle savedInstanceState) {
        return inflater.inflate(R.layout.fragment_simple, container, false);
    }
}
```

### 3. Жизненный цикл фрагмента (20 мин)
- Пошаговый обзор каждого метода.  
- Взаимодействие с жизненным циклом активности.  
- Как сохранять и восстанавливать состояние с помощью `onSaveInstanceState`.  
- Диаграмма жизненного цикла (текстовое описание).

### 4. Управление фрагментами (20 мин)
- Добавление, удаление и замена фрагментов через `FragmentTransaction`.  
- Использование `addToBackStack` для навигации.  
- Пример: переключение между двумя фрагментами.

```java
FragmentTransaction ft = getSupportFragmentManager().beginTransaction();
ft.replace(R.id.fragment_container, new SecondFragment());
ft.addToBackStack(null);
ft.commit();
```

### 5. Передача данных между фрагментами (10 мин)
- Использование `Bundle` для аргументов.  
- Паттерн `newInstance`.  
- Пример передачи строки в фрагмент.

```java
public static MyFragment newInstance(String title) {
    MyFragment f = new MyFragment();
    Bundle args = new Bundle();
    args.putString("title", title);
    f.setArguments(args);
    return f;
}
```

### 6. Продвинутые техники (15 мин)
- Вложенные фрагменты (nested fragments).  
- `ViewModel` и `LiveData` для обмена данными между фрагментами.  
- Анимации переходов: `setCustomAnimations`.  

## Примеры и иллюстрации
- **Диаграмма жизненного цикла** (текстовое описание, заменено на таблицу).

| Событие | Описание |
|---------|----------|
| `onAttach` | Фрагмент прикрепляется к активности. |
| `onCreate` | Инициализация данных. |
| `onCreateView` | Создание UI. |
| `onActivityCreated` | Доступ к элементам активности. |
| `onStart` | Фрагмент видим, но не активен. |
| `onResume` | Полная активность. |
| `onPause` | Фрагмент временно неактивен. |
| `onStop` | Фрагмент скрыт. |
| `onDestroyView` | Удаление UI. |
| `onDestroy` | Очистка ресурсов. |
| `onDetach` | Отсоединение от активности. |

- **Код с `FragmentManager`**:

```java
FragmentManager fm = getSupportFragmentManager();
FragmentTransaction ft = fm.beginTransaction();
ft.add(R.id.container, new ExampleFragment());
ft.commit();
```

## Выводы по лекции
- Фрагменты позволяют создавать модульные и переиспользуемые UI-компоненты.  
- Понимание жизненного цикла критично для корректной работы и избежания утечек памяти.  
- Управление фрагментами через `FragmentManager` и `FragmentTransaction` обеспечивает гибкую навигацию.  
- Передача данных и сохранение состояния — ключевые практики при работе с фрагментами.
