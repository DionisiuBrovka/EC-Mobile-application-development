# Жизненный цикл активности. Графический интерфейс пользователя

## Цели и задачи лекции
- Понять, что такое жизненный цикл активности в контексте приложений на Python.  
- Ознакомиться с ключевыми этапами создания графического интерфейса пользователя (GUI).  
- Научиться реализовывать простые GUI‑приложения с помощью библиотеки Tkinter.  
- Освоить практические навыки отладки и тестирования GUI‑приложений.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| Активность | Логическое отделение части программы, отвечающей за пользовательский ввод и вывод. |
| Жизненный цикл | Последовательность состояний, через которые проходит активность от создания до завершения. |
| Событие | Внешний сигнал (нажатие кнопки, перемещение мыши), инициирующий обработку. |
| Обработчик событий | Функция, привязанная к событию, выполняющая конкретную задачу. |
| Главное окно | Корневой контейнер GUI, в котором размещаются все остальные элементы. |
| Виджет | Элемент интерфейса (кнопка, поле ввода, метка). |
| Конвейер событий | Механизм, при котором события обрабатываются последовательно в порядке их возникновения. |
| Схема управления | Модель взаимодействия пользователя и программы (например, MVC). |
| Отладка GUI | Процесс выявления и исправления ошибок, связанных с пользовательским интерфейсом. |
| Тестирование GUI | Проверка корректности работы интерфейса с использованием автоматических или ручных методов. |

## Основное содержание

### 1. Введение в жизненный цикл активности (10 мин)
- Что такое активность в приложениях.  
- Сравнение с веб‑страницами и мобильными приложениями.  
- Роль жизненного цикла в управлении ресурсами.

### 2. Структура GUI‑приложения на Python (15 мин)
- Выбор библиотеки: Tkinter, PyQt, wxPython.  
- Обзор Tkinter как стандартного инструмента.  
- Создание главного окна (`tk.Tk()`) и настройка свойств.

```python
import tkinter as tk

root = tk.Tk()
root.title("Пример GUI")
root.geometry("300x200")
```

### 3. Создание и размещение виджетов (15 мин)
- Основные виджеты: `Button`, `Label`, `Entry`, `Frame`.  
- Методы размещения: `pack()`, `grid()`, `place()`.  
- Пример: интерфейс калькулятора.

```python
frame = tk.Frame(root)
frame.pack(pady=10)

btn = tk.Button(frame, text="Нажми меня")
btn.pack()
```

### 4. Обработка событий (15 мин)
- Что такое событие и как его привязать к обработчику.  
- Методы `bind()` и `command`.  
- Пример: изменение текста при нажатии кнопки.

```python
def on_click():
    label.config(text="Кнопка нажата")

btn = tk.Button(root, text="Нажми", command=on_click)
```

### 5. Жизненный цикл GUI‑приложения (10 мин)
- Фазы: инициализация, цикл обработки событий, завершение.  
- Как управлять ресурсами (открытие/закрытие файлов, сетевых соединений).  
- Пример: корректное закрытие приложения.

```python
def on_close():
    # Очистка ресурсов
    root.destroy()

root.protocol("WM_DELETE_WINDOW", on_close)
```

### 6. Отладка и тестирование GUI (10 мин)
- Инструменты: `pdb`, `logging`, `unittest` с `tkinter`.  
- Ручное тестирование: проверка всех виджетов и событий.  
- Автоматическое тестирование: использование `pytest-qt` (если используется PyQt).  

## Примеры и иллюстрации
- **Пример простого окна с кнопкой**:

```python
import tkinter as tk

def greet():
    print("Привет, мир!")

root = tk.Tk()
root.title("Пример")

btn = tk.Button(root, text="Поздороваться", command=greet)
btn.pack(padx=20, pady=20)

root.mainloop()
```

- **Схема жизненного цикла**:

```
Инициализация → Событие → Обработчик → Обновление UI → Завершение
```

## Выводы по лекции

- Эффективная обработка событий и правильное управление ресурсами критичны для качественного пользовательского опыта.
