# Составление заключения и списка использованных источников

## Цели и задачи лекции
- Познакомить студентов с принципами написания заключения в научно-исследовательских работах.  
- Рассмотреть стандарты оформления списка использованных источников.  
- Научить применять правила цитирования и библиографического оформления в Python‑проекте.  
- Предоставить практические задания по созданию заключения и библиографии.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| Заключение | Сводный раздел, подытоживающий результаты исследования. |
| Список использованных источников | Перечень всех материалов, на которые ссылается работа. |
| Синтаксис цитирования | Правила оформления ссылок в тексте и библиографии. |
| BibTeX | Формат хранения библиографических записей, часто используемый в LaTeX. |
| Markdown | Язык разметки, применяемый в Obsidian для оформления текста. |
| DOI | Уникальный цифровой идентификатор научной публикации. |
| Ссылка на источник | Указание конкретного места (строки, секции) в источнике. |
| Рецензирование | Процесс проверки работы на соответствие академическим стандартам. |
| APA | Стиль цитирования Американской психологической ассоциации. |
| MLA | Стиль цитирования Американского литературного союза. |

## Основное содержание

### 1. Структура заключения (20 мин)
- Введение: краткое напоминание целей исследования.  
- Основные результаты: перечисление ключевых выводов.  
- Ограничения исследования: что не удалось учесть.  
- Перспективы дальнейших работ: направления для будущих исследований.

#### Пример структуры в Markdown
```markdown
## Заключение

В данной работе исследованы...

### Основные результаты
1. ...
2. ...

### Ограничения
...

### Перспективы
...
```

### 2. Форматирование библиографического списка (30 мин)
- Общие правила: фамилия автора, инициалы, год, название, издательство, DOI.  
- Стили APA и MLA: основные отличия.  
- Использование Markdown‑таблиц для упорядочивания записей.

#### Пример APA в Markdown
```markdown
## Список использованных источников

| Автор | Год | Тема | DOI |
|-------|-----|------|-----|
| Иванов И. | 2021 | Python для начинающих | 10.1000/xyz123 |
| Петров П. | 2019 | Алгоритмы и структуры данных | 10.1000/abc456 |
```

### 3. Автоматизация создания библиографии в Python (25 мин)
- Библиотека `bibtexparser` для чтения `.bib` файлов.  
- Генерация Markdown‑таблицы из BibTeX.  
- Интеграция с Obsidian через плагины.

#### Код‑пример
```python
import bibtexparser
from pathlib import Path

bib_file = Path('references.bib')
with open(bib_file) as bibtex_file:
    bib_database = bibtexparser.load(bibtex_file)

for entry in bib_database.entries:
    print(f"| {entry['author']} | {entry['year']} | {entry['title']} | {entry.get('doi', '—')} |")
```

### 4. Практическое упражнение (15 мин)
- Студенты создают собственное заключение по теме проекта.  
- Форматируют список источников в стиле APA.  
- Проверяют оформление с помощью онлайн‑инструмента (не упоминать ссылки).

## Выводы по лекции
- Заключение должно быть лаконичным, но информативным.  
- Библиография требует строгого соблюдения стиля цитирования.  
- Markdown позволяет гибко оформлять списки и таблицы.  
- Автоматизация через Python упрощает работу с большим количеством источников.
