# Защита курсового проекта

## Цели и задачи лекции
- Понять основные этапы подготовки и проведения защиты курсового проекта.
- Освоить ключевые требования к оформлению и представлению результатов.
- Научиться аргументировать решения и отвечать на вопросы аудитории.
- Приобрести навыки работы с презентационными средствами и средствами презентации кода.

## Ключевые понятия
| Понятие | Определение |
|---------|-------------|
| **Курсовой проект** | Практическая работа, демонстрирующая умение применять изученные навыки. |
| **Защита проекта** | Формальное представление результатов и обоснование решений перед комиссией. |
| **План защиты** | Схема, включающая структуру презентации, временные рамки и ключевые моменты. |
| **Комиссия** | Группа преподавателей, оценивающих проект по установленным критериям. |
| **Критерии оценки** | Набор требований к содержанию, технике выполнения и защите проекта. |
| **Промежуточный отчёт** | Документ, отражающий ход работы и промежуточные результаты. |
| **Формат презентации** | Слайдовый документ, PDF или веб‑страница, содержащая визуальные и текстовые материалы. |
| **Техническая демонстрация** | Прямое показание работы программы в реальном времени. |
| **Ответы на вопросы** | Период во время защиты, когда комиссия задаёт уточняющие вопросы. |
| **Рефлексия** | Самоанализ выполненной работы и выявление областей для улучшения. |

## Основное содержание

### 1. Подготовка к защите (30 минут)

#### 1.1 Сбор и систематизация материалов
- Перечисление всех файлов, документации, кода.
- Создание единой структуры проекта (папки, README.md).

#### 1.2 Создание презентации
- Выбор шаблона (Google Slides, PowerPoint, Obsidian Slides).
- Структура слайдов: цель, задачи, методика, результаты, выводы, вопросы.

#### 1.3 Тестирование демонстрации
- Проверка работы кода на разных средах.
- Подготовка резервного варианта (скриншоты, видео).

### 2. Оформление проекта (20 минут)

#### 2.1 Структура отчёта
- Введение, цели, методы, результаты, выводы, список использованных источников.

#### 2.2 Кодирование и стиль
- Соблюдение PEP‑8, комментарии, docstring.
- Использование виртуального окружения.

#### 2.3 Литература и источники
- Список книг, статей, официальной документации.

### 3. Проведение защиты (30 минут)

#### 3.1 Презентация
- Чёткое изложение содержания, соблюдение тайминга.
- Использование графиков, таблиц, диаграмм.

#### 3.2 Демонстрация кода
```python
def greet(name: str) -> str:
    """Возвращает приветственное сообщение."""
    return f"Привет, {name}!"

print(greet("Андрей"))
```

#### 3.3 Ответы на вопросы
- Подготовка к типичным вопросам (обоснование выбора технологий, сложности, альтернативы).

#### 3.4 Итоговый рефлексивный блок
- Краткое подведение итогов и выводов, рекомендации по дальнейшему развитию.
