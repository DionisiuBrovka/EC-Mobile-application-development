# Обработка исключительных ситуаций

## Цели и задачи лекции
1. Познакомить студентов с понятием исключений в Python.  
2. Выяснить правила работы с конструкциями `try`, `except`, `else`, `finally`.  
3. Научить создавать собственные исключения и использовать их в программах.  
4. Показать, как корректно обрабатывать ошибки и вести логирование.

## Ключевые понятия
- **Исключение** – объект, указывающий на ошибку во время выполнения.  
- **Трассировка (traceback)** – стек вызовов, показывающий путь до ошибки.  
- **Блок `try`** – участок кода, где может возникнуть исключение.  
- **Блок `except`** – обработчик конкретного типа исключения.  
- **Блок `else`** – выполняется, если в `try` исключения не возникло.  
- **Блок `finally`** – выполняется всегда, независимо от результата.  
- **`raise`** – оператор, генерирующий исключение вручную.  
- **`assert`** – условие, генерирующее `AssertionError` при ложности.  
- **Пользовательские исключения** – подклассы `Exception`, созданные пользователем.  
- **Логирование** – запись информации об ошибках в файл или консоль.

## Основное содержание

### 1. Введение в исключения (10 мин)
- Что такое исключения и почему их важно обрабатывать.  
- Сравнение с обработкой ошибок в других языках.  

### 2. Структура блока `try / except` (15 мин)
- Синтаксис:  
  ```python
  try:
      # код
  except SomeError as e:
      # обработка
  ```
- Обработка нескольких типов ошибок.  
- Перехват всех ошибок (`except Exception:`).  

### 3. Блок `else` и `finally` (10 мин)
- Когда и зачем использовать `else`.  
- Обязательная роль `finally` при работе с ресурсами.  
  ```python
  try:
      f = open('file.txt')
  except FileNotFoundError:
      print('Файл не найден')
  else:
      print('Файл открыт')
  finally:
      f.close()
  ```

### 4. Генерация исключений (`raise`) (10 мин)
- Как вручную вызвать ошибку.  
- Параметры `raise` и передача собственного сообщения.  

### 5. Создание пользовательских исключений (10 мин)
- Наследование от `Exception`.  
- Добавление дополнительных атрибутов.  
  ```python
  class ValidationError(Exception):
      def __init__(self, field, message):
          self.field = field
          super().__init__(message)
  ```

### 6. Логирование ошибок (10 мин)
- Модуль `logging`.  
- Пример записи ошибки в файл.  
  ```python

  logging.basicConfig(filename='app.log', level=logging.ERROR)

      1/0
  ```

## Примеры и иллюстрации

```python
# Пример обработки нескольких ошибок
try:
    number = int(input("Введите число: "))
    result = 10 / number
except ValueError:
    print("Ошибка: введено не число")
except ZeroDivisionError:
    print("Ошибка: деление на ноль")
else:
    print(f"Результат: {result}")
finally:
    print("Блок finally всегда выполняется")
```

```python
# Пользовательское исключение
class NegativeNumberError(Exception):
    pass

def sqrt(x):
    if x < 0:
        raise NegativeNumberError("Нельзя брать корень из отрицательного числа")
    return x ** 0.5

try:
    print(sqrt(-9))
except NegativeNumberError as e:
    print(e)
```

## Выводы по лекции
- Исключения позволяют отделить ошибочный код от основной логики.  
- Конструкции `try / except / else / finally` дают гибкий контроль над обработкой ошибок.  
- Создание пользовательских исключений повышает читаемость и управляемость кода.  
- Логирование является неотъемлемой частью надёжной разработки.
