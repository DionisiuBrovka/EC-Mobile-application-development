# Курсоры. Получение данных и выполнение в фоновом режиме

Курсоры и фоновые задачи – это фундаментальные инструменты, позволяющие обрабатывать большие объёмы данных, интегрировать внешние сервисы и строить масштабируемые приложения. В индустрии они применяются в ETL‑процессах, аналитике реального времени, чат‑ботах и микросервисах. Понимание их работы открывает путь к проектированию высокопроизводительных систем и к участию в крупных инженерных проектах.

## Цели и задачи

1. Освоить концепцию курсоров в Python и их связь с базами данных и потоками данных.  
2. Понять принципы асинхронного и многопоточного выполнения задач.  
3. Научиться строить эффективные пайплайны для извлечения, трансформации и загрузки данных (ETL).  
4. Развивать аналитическое мышление через оценку производительности и отказоустойчивости систем.  
5. Подготовить студентов к реальным задачам в области больших данных и DevOps.

## Ключевые понятия

| Термин | Расширенное определение |
|--------|------------------------|
| **Курсор** | Указатель, управляющий чтением строк из результата запроса. Позволяет обрабатывать данные пакетами, не загружая их полностью в память. |
| **Поток данных** | Последовательность элементов, генерируемых и потребляемых в реальном времени. Используется для обработки событий, логов и сетевого трафика. |
| **Асинхронность** | Модель выполнения, где операции (IO‑операции, запросы) запускаются без блокировки потока, позволяя обрабатывать несколько задач одновременно. |
| **Многопоточность** | Использование нескольких потоков исполнения в одном процессе, что повышает параллелизм, но требует синхронизации и избегания гонок. |
| **Batch Processing** | Обработка данных пакетами фиксированного размера. Эффективно для больших объёмов, когда можно использовать индексацию и кеширование. |
| **Streaming Processing** | Постоянное потребление и обработка данных в реальном времени, часто реализуется через очереди и брокеры сообщений. |
| **Task Queue** | Очередь задач, в которой фоновые воркеры выполняют тяжелые операции, освобождая основной поток от блокирующих вызовов. |
| **Concurrency** | Способность системы обрабатывать несколько операций одновременно, не обязательно параллельно. |
| **Throughput** | Скорость обработки данных (строк/секунды). Ключевой метрика при проектировании систем. |
| **Latency** | Время от начала запроса до получения ответа. Важна при интерактивных и реального‑временных приложениях. |

## Примеры и иллюстрации

```python
# Пример курсора в sqlite3
import sqlite3

conn = sqlite3.connect('example.db')
cur = conn.cursor()
cur.execute('SELECT * FROM users')

for row in cur:
    print(row)  # Обрабатываем построчно, не загружая всё в память
```

```python
# Асинхронный запрос к API с использованием aiohttp
import asyncio
import aiohttp

async def fetch(session, url):
    async with session.get(url) as resp:
        return await resp.json()

async def main():
    urls = ['https://api.github.com/users/octocat'] * 1000
    async with aiohttp.ClientSession() as session:
        tasks = [fetch(session, u) for u in urls]
        results = await asyncio.gather(*tasks)
    print(f'Fetched {len(results)} responses')

asyncio.run(main())
```

**Кейс из индустрии**  
Международный финтех‑стартап использует `Celery` с брокером RabbitMQ для обработки тысяч транзакций в секунду. Курсоры в PostgreSQL позволяют читать данные порциями, а `Celery` распределяет задачи по воркерам, обеспечивая отказоустойчивость и масштабируемость.
