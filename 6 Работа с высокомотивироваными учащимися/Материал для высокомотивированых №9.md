# Фрагменты. Жизненный цикл фрагментов  

Фрагменты — это небольшие, переиспользуемые компоненты пользовательского интерфейса, которые позволяют создавать модульные, масштабируемые приложения. Они критичны для мобильной разработки (Android, Flutter, React‑Native) и для веб‑приложений, где динамическая перестройка UI повышает производительность и упрощает тестирование. Понимание жизненного цикла фрагментов даёт студентам навыки проектирования гибких архитектур, управления состоянием и оптимизации пользовательского опыта.

## Цели и задачи  

1. **Освоить архитектурные принципы** работы фрагментов и их взаимодействие с активностями/компонентами.  
2. **Понять этапы жизненного цикла** фрагментов, включая инициализацию, отображение, сохранение состояния и уничтожение.  
3. **Развить навыки управления состоянием** через ViewModel, SavedStateHandle и другие паттерны.  
4. **Анализировать и сравнивать** фрагменты с другими UI‑компонентами (например, `View` в Android, `Widget` в Flutter).  
5. **Построить исследовательскую задачу**: оценить влияние разных стратегий жизненного цикла на производительность и UX.

## Ключевые понятия  

| Термин | Расширенное определение | Зачем нужно |
|--------|------------------------|-------------|
| **Fragment** | Модульный UI‑элемент, объединяющий разметку, логику и взаимодействие. | Позволяет переиспользовать код и динамически менять UI. |
| **Lifecycle** | Набор методов (`onCreate`, `onStart`, `onResume`, `onPause`, `onStop`, `onDestroy`) и состояний, через которые проходит фрагмент. | Обеспечивает предсказуемое управление ресурсами и состоянием. |
| **ViewModel** | Компонент, сохраняющий данные UI вне UI‑элементов, выживающий при конфигурационных изменениях. | Защищает данные от потери при повороте экрана, облегчает тестирование. |
| **SavedStateHandle** | Хранилище для сохранения состояния фрагмента в `Bundle`. | Позволяет восстановить UI‑состояние без прямого доступа к `Bundle`. |
| **FragmentManager** | Менеджер, отвечающий за добавление, удаление и замену фрагментов внутри контейнера. | Управляет транзакциями, обеспечивает обратную совместимость. |
| **FragmentTransaction** | Операция, описывающая изменения в стеке фрагментов (add, replace, remove). | Позволяет анимировать переходы и управлять историей. |
| **Navigation Component** | Библиотека для упрощения навигации между фрагментами. | Делает переходы декларативными, управляет back‑stack. |
| **Fragment Result API** | Механизм обмена данными между фрагментами через `setFragmentResultListener`. | Избегает прямой зависимости и повышает модульность. |
| **Fragment Lifecycle Owner** | Объект, который управляет подписками на LiveData в соответствии с жизненным циклом фрагмента. | Предотвращает утечки памяти и обновления после уничтожения UI. |
| **Lazy Loading** | Загрузка ресурсов только при необходимости. | Оптимизирует использование памяти и ускоряет старт приложения. |

## Примеры и иллюстрации  

```kotlin
class SampleFragment : Fragment(R.layout.fragment_sample) {

    private val viewModel: SampleViewModel by viewModels()
    private var _binding: FragmentSampleBinding? = null
    private val binding get() = _binding!!

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        Log.d("Lifecycle", "onCreate")
    }

    override fun onCreateView(
        inflater: LayoutInflater,
        container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View {
        Log.d("Lifecycle", "onCreateView")
        _binding = FragmentSampleBinding.inflate(inflater, container, false)
        return binding.root
    }

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        Log.d("Lifecycle", "onViewCreated")
        binding.button.setOnClickListener {
            viewModel.loadData()
        }
        viewModel.data.observe(viewLifecycleOwner) { data ->
            binding.textView.text = data
        }
    }

    override fun onDestroyView() {
        super.onDestroyView()
        Log.d("Lifecycle", "onDestroyView")
        _binding = null
    }
}
```

### Нечёткий кейс: динамическая карта местоположений  
В мессенджере, где пользователь может открывать карту в фрагменте, но при смене экрана приложение должно быстро переключаться без полной перезагрузки карты. Использование `FragmentManager` с `add` и `detach/attach` вместо `replace` позволяет сохранить состояние карты и ускорить переход.

### Сравнительный анализ  
| Платформа | Жизненный цикл | Состояние | Управление навигацией |
|-----------|----------------|-----------|-----------------------|
| Android (Fragment) | onCreate → onStart → onResume → onPause → onStop → onDestroy | ViewModel, SavedStateHandle | FragmentManager, Navigation Component |
| Flutter (Widget) | build → initState → dispose | Provider, Bloc | Navigator, Routes |
| React Native (Component) | componentDidMount → componentDidUpdate → componentWillUnmount | Redux, Context | React Navigation |

## Проектные идеи  

1. **Модульный UI‑конструктор**: разработать приложение, где пользователь может динамически добавлять/удалять фрагменты из списка, сохраняя состояние при перезапуске.  
2. **Система уведомлений с фрагментами**: реализовать фрагмент, который отображает список уведомлений, используя `LazyColumn` и `ViewModel` для управления данными.  
3. **Анализ производительности**: собрать метрики по времени создания и уничтожения фрагментов при различных сценариях (переходы, конфигурационные изменения) и предложить оптимизации.

## Выводы  

Фрагменты являются фундаментом гибкой архитектуры UI, позволяя разбивать приложения на независимые, переиспользуемые части. Понимание их жизненного цикла, методов управления состоянием и навигации открывает возможности для создания масштабируемых и поддерживаемых приложений. В дальнейшем стоит изучить интеграцию фрагментов с современными библиотеками управления состоянием (Jetpack Compose, Redux) и исследовать паттерны реактивного программирования.
