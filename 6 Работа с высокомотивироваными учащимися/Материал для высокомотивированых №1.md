# Работа с базой данных SQLite

SQLite – это встроенная реляционная база данных, которая не требует отдельного сервера и хранит данные в одном файле. Благодаря своей лёгкости, простоте развертывания и высокой производительности, SQLite используется в мобильных приложениях, встраиваемых системах, а также как «пробный» слой для прототипирования и обучения. Для будущих инженеров программного обеспечения важно понимать, как эффективно работать с SQLite, ведь многие современные проекты требуют быстрых, автономных решений для локального хранения данных.

## Цели и задачи

1. **Освоить базовый синтаксис SQL в контексте SQLite** – создание таблиц, индексов, запросы с `JOIN`, `GROUP BY`, `HAVING`.
2. **Развить умения проектировать схему базы данных** – нормализация, выбор подходящих типов данных, ограничений целостности.
3. **Анализировать и оптимизировать запросы** – чтение планов выполнения, индексы, использование `EXPLAIN QUERY PLAN`.
4. **Интегрировать SQLite с Python** – работа через `sqlite3`, `SQLAlchemy`, ORM‑слои, миграции.
5. **Понимать практические сценарии применения** – от мобильных приложений до небольших сервисов и прототипов.

## Ключевые понятия

| Термин | Расширенное определение |
|--------|------------------------|
| **Транзакция** | Единица работы, которая выполняется полностью или не выполняется вовсе. В SQLite транзакции гарантируют атомарность, целостность и изоляцию, что критично при одновременном доступе. |
| **Индексы** | Структуры данных, ускоряющие поиск. В SQLite индексы поддерживают `UNIQUE`, `PRIMARY KEY`, `FOREIGN KEY` и могут быть созданными на выражениях. |
| **Нормализация** | Процесс упорядочивания данных для минимизации избыточности. Нормальные формы (2NF, 3NF) позволяют избежать аномалий обновления. |
| **VACUUM** | Команда, которая пересоздаёт базу, освобождая место и упорядочивая данные. Полезна после массового удаления строк. |
| **PRAGMA** | Специальные команды, управляющие внутренним состоянием SQLite (например, `PRAGMA foreign_keys = ON`). |
| **BLOB** | Binary Large Object – тип, хранящий произвольные бинарные данные (изображения, аудио). |
| **Внешние ключи** | Ограничения, обеспечивающие целостность данных между таблицами. В SQLite их поддержка включается явно. |
| **Virtual Table** | Таблица, реализованная в пользовательском коде (например, FTS5 – полнотекстовый поиск). |
| **Triggers** | Хранимые процедуры, автоматически выполняющиеся при изменении таблиц. |
| **Connection Pooling** | Управление несколькими соединениями для повышения производительности. В SQLite обычно не требуется из‑за однопоточного доступа, но полезно в многопоточных приложениях. |

## Примеры и иллюстрации

```sql
-- Создание таблицы авторов
CREATE TABLE authors (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    birth_year INTEGER
);

-- Создание таблицы книг с внешним ключом
CREATE TABLE books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    author_id INTEGER,
    published_year INTEGER,
    FOREIGN KEY(author_id) REFERENCES authors(id)
);
```

```python
import sqlite3

# Подключение и создание таблиц
conn = sqlite3.connect('library.db')
c = conn.cursor()
c.executescript('''
    PRAGMA foreign_keys = ON;
    CREATE TABLE IF NOT EXISTS authors (...);
    CREATE TABLE IF NOT EXISTS books (...);
''')
conn.commit()
```

> **Кейс из индустрии**  
> В мобильном приложении «Книга дня» используется SQLite для кэширования информации о книгах и пользователях. Благодаря встроенному индексу по `author_id` поиск книг по автору занимает миллисекунды даже при 10 000 записях.

## Проектные идеи

1. **Прототип мобильного приложения для заметок** – реализовать локальное хранение заметок и тегов с полнотекстовым поиском.
2. **Микросервис «Калькулятор налогов»** – хранить таблицы ставок и правил, обрабатывать запросы через REST API, использовать SQLite как быстрый «кеш» для часто запрашиваемых данных.
3. **Библиотека анализа логов** – собрать логи из разных источников в SQLite, выполнять агрегацию и визуализацию в Jupyter Notebook.

## Выводы

SQLite предоставляет мощный, но простой инструмент для локального хранения данных, который легко интегрируется с Python. Понимание его особенностей, таких как транзакции, индексы и расширенные типы, открывает путь к созданию надёжных, быстрых и масштабируемых приложений. Для дальнейшего развития стоит изучить продвинутые темы: виртуальные таблицы, асинхронное взаимодействие и миграции схемы через Alembic.
