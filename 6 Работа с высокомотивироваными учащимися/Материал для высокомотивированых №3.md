# Методы и конструкторы

В современной разработке Python становится критически важным не только писать работающий код, но и строить его с использованием надёжных, расширяемых и читаемых паттернов. Методы и конструкторы — фундаментальные механизмы, позволяющие управлять поведением объектов, обеспечивать инкапсуляцию и поддерживать высокую модульность. В этой лекции мы исследуем, как эти инструменты применяются в реальных проектах, какие преимущества они дают и как их можно комбинировать для решения сложных задач.

## Цели и задачи

1. **Понять роль методов и конструкторов в объектно‑ориентированном программировании** и их влияние на архитектуру кода.  
2. **Разобрать ключевые паттерны проектирования** (Factory, Builder, Singleton, Decorator), их применение и ограничения.  
3. **Развить аналитическое мышление** через сравнение различных подходов к решению одной и той же задачи.  
4. **Освоить практические навыки** написания чистого, тестируемого кода с использованием современных библиотек и инструментов.  
5. **Сформировать идеи для исследовательских проектов** и стартапов, основанных на применении методов и конструкторов.

## Ключевые понятия

| Термин | Расширенное определение | Зачем это нужно |
|--------|-------------------------|------------------|
| **Метод** | Функция, привязанная к объекту, выполняющая операции над его состоянием. | Позволяет инкапсулировать логику, обеспечивая интерфейс для взаимодействия с объектом. |
| **Конструктор (`__init__`)** | Специальный метод, вызываемый при создании экземпляра класса, инициализирующий атрибуты. | Гарантирует, что объект находится в корректном состоянии сразу после создания. |
| **Фабрика (Factory)** | Паттерн, который определяет интерфейс создания объектов, но позволяет подклассам решать, какой класс инстанцировать. | Избегает жесткой зависимости от конкретных классов и упрощает расширяемость. |
| **Builder** | Паттерн, позволяющий пошагово строить сложный объект, разделяя его создание и представление. | Упрощает создание объектов с множеством опций и конфигураций. |
| **Singleton** | Паттерн, обеспечивающий существование только одного экземпляра класса в приложении. | Контролирует доступ к общим ресурсам (например, логгер, пул соединений). |
| **Decorator** | Паттерн, добавляющий функциональность объекту динамически, не меняя его интерфейс. | Позволяет гибко расширять поведение без наследования. |
| **Инкапсуляция** | Сокрытие внутреннего состояния объекта от внешнего мира. | Защищает данные от некорректных изменений и упрощает сопровождение кода. |
| **Абстракция** | Выделение существенных характеристик объекта, игнорируя детали реализации. | Облегчает понимание и взаимодействие с кодом. |
| **Переопределение методов** | Замена реализации метода в подклассе. | Позволяет реализовать полиморфизм и адаптировать поведение. |
| **Классовый метод (`@classmethod`)** | Метод, принимающий класс как первый аргумент вместо экземпляра. | Полезен для фабричных методов и альтернативных конструкторов. |

## Примеры и иллюстрации

```python
# 1. Фабрика для парсеров
from abc import ABC, abstractmethod

class Parser(ABC):
    @abstractmethod
    def parse(self, data: str):
        pass

class JsonParser(Parser):
    def parse(self, data: str):
        import json
        return json.loads(data)

class CsvParser(Parser):
    def parse(self, data: str):
        import csv
        from io import StringIO
        return list(csv.DictReader(StringIO(data)))

class ParserFactory:
    @staticmethod
    def create_parser(file_type: str) -> Parser:
        if file_type.lower() == 'json':
            return JsonParser()
        elif file_type.lower() == 'csv':
            return CsvParser()
        else:
            raise ValueError('Unsupported format')

# Использование
factory = ParserFactory()
parser = factory.create_parser('csv')
records = parser.parse("name,age\nAlice,30\nBob,25")
print(records)
```

```python
# 2. Builder для конфигурации сервера
class Server:
    def __init__(self, host, port, ssl=False, timeout=30):
        self.host = host
        self.port = port
        self.ssl = ssl
        self.timeout = timeout

class ServerBuilder:
    def __init__(self):
        self._host = 'localhost'
        self._port = 8080
        self._ssl = False
        self._timeout = 30

    def host(self, host):
        self._host = host
        return self

    def port(self, port):
        self._port = port
        return self

    def ssl(self, ssl=True):
        self._ssl = ssl
        return self

    def timeout(self, timeout):
        self._timeout = timeout
        return self

    def build(self):
        return Server(self._host, self._port, self._ssl, self._timeout)

# Конфигурация
server = (
    ServerBuilder()
    .host('example.com')
    .port(443)
    .ssl()
    .timeout(60)
    .build()
)
print(server.__dict__)
```

```python
# 3. Singleton логгер
import logging

class LoggerSingleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super().__new__(cls)
            logging.basicConfig(level=logging.INFO)
            cls._instance.logger = logging.getLogger('app')
        return cls._instance

# Использование
log1 = LoggerSingleton().logger
log2 = LoggerSingleton().logger
log1.info("First logger instance")
assert log1 is log2  # True
```

```python
# 4. Декоратор кэширования
from functools import wraps
from time import time

def simple_cache(func):
    cache = {}
    @wraps(func)
    def wrapper(*args):
        if args in cache:
            return cache[args]
        result = func(*args)
        cache[args] = result
        return result
    return wrapper

@simple_cache
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)

print(fib(35))  # Быстрое выполнение благодаря кэшу
```

1. **Фреймворк для динамической генерации API**: Используя фабрики и декораторы, построить библиотеку, позволяющую описывать модели данных и автоматически генерировать REST‑эндпоинты с валидацией и сериализацией.

## Выводы

Методы и конструкторы – это не просто синтаксические конструкции, а мощные инструменты, позволяющие строить гибкую, поддерживаемую и масштабируемую архитектуру. Паттерны Factory, Builder, Singleton и Decorator дают готовые решения типичных проблем проектирования. Понимание их принципов и умение применять в реальных задачах открывает двери к более высоким уровням профессионализма и креативности в IT.
