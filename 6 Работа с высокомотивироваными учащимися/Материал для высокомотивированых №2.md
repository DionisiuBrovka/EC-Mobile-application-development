# Массивы

Массивы (или списки в Python) являются фундаментальным типом данных, позволяющим хранить упорядоченную коллекцию элементов. Их правильное понимание открывает путь к эффективному управлению памятью, построению алгоритмов и работе с большими данными. В индустрии массивы встречаются везде: от баз данных и аналитики до машинного обучения и систем реального времени.

## Цели и задачи

1. Понять внутреннюю структуру массивов и их отличие от других коллекций (словарей, кортежей, генераторов).  
2. Освоить методы создания, модификации и доступа к элементам массивов с учётом производительности.  
3. Разработать навыки анализа сложностей операций над массивами и выбора оптимальных структур в конкретных задачах.  
4. Научиться применять массивы в решении реальных задач: обработка данных, графики, алгоритмы поиска и сортировки.  
5. Воспитать исследовательское мышление: уметь задавать вопросы о том, почему и как использовать массивы в разных контекстах.

## Ключевые понятия

| Термин | Расширенное определение |
|--------|------------------------|
| **Массив (list)** | Упорядоченная коллекция, поддерживающая произвольный доступ к элементам по индексу. Используется, когда важна скорость чтения/записи и гибкость изменения размера. |
| **Кортеж (tuple)** | Неизменяемый массив. Предназначен для хранения данных, которые не должны изменяться; обеспечивает безопасность и может быть использован как ключ в словаре. |
| **Индексация** | Способ обращения к элементу по позиции; в Python начинается с нуля и поддерживает отрицательные индексы, что позволяет обращаться к элементам с конца. |
| **Срез (slice)** | Подмножество массива, создаваемое копией выбранных элементов. Позволяет быстро извлекать диапазоны без изменения исходного массива. |
| **Динамическое расширение** | Механизм, при котором массив автоматически увеличивает свой размер при добавлении элементов, обычно с запасом памяти для снижения частоты перераспределения. |
| **Статический массив** | Массив фиксированного размера; в Python это `array.array` из модуля `array` или `numpy.ndarray` для числовых данных. |
| **Собственная реализация** | Создание кастомного массива (например, на основе `list` и `ctypes`) для специфических требований, таких как типизированные данные или ограничения памяти. |
| **Алгоритм сортировки** | Методы упорядочивания элементов (quicksort, mergesort, heapsort) и их влияние на временную и пространственную сложность при работе с массивами. |
| **Алгоритм поиска** | Бинарный поиск, линейный поиск и их применение в зависимости от сортировки массива. |
| **Память и кэш** | Понимание того, как расположение элементов в памяти влияет на производительность из-за кэш-локальности. |

## Примеры и иллюстрации

```python
# Быстрый доступ к элементу
arr = list(range(10_000_000))
print(arr[5_000_000])  # 5000000

# Срезы и копирование
sub = arr[100:200]  # 100 элементов

# Быстрая сортировка с использованием heapq
import heapq
data = [5, 3, 8, 1, 9]
sorted_data = [heapq.heappop(data) for _ in range(len(data))]
print(sorted_data)  # [1, 3, 5, 8, 9]
```

**Кейс из индустрии:**  
В системах рекомендаций часто используют массивы для хранения пользовательских профилей и предпочтений. Оптимизация доступа к элементам позволяет снизить задержку до миллисекунд, что критично для реального времени.

**Небольшой эксперимент:**  
```python
import time
lst = []
start = time.time()
for i in range(1_000_000):
    lst.append(i)
print("Append:", time.time() - start)

start = time.time()
for i in range(1_000_000):
    _ = lst[i]
print("Index:", time.time() - start)
```

Понимание их внутренней структуры, алгоритмов обработки и памяти позволяет строить более надёжные и быстрые системы.
