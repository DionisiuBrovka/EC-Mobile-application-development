# Виджеты-списки и привязка данных. Меню. Диалоговые окна

В современном программировании пользовательский интерфейс становится одним из ключевых факторов успешности продукта. Виджеты-списки, привязка данных, контекстные меню и диалоговые окна позволяют создавать интуитивно понятные, гибкие и масштабируемые приложения. Для инженеров, работающих с Python, знание этих концепций открывает двери к разработке высококачественных GUI, автоматизации процессов, аналитических панелей и интерактивных прототипов.

## Цели и задачи

1. **Освоить принципы построения списковых виджетов** (TreeView, ListBox, DataGrid) и их связь с источниками данных.  
2. **Понимать механизмы привязки данных** (Data Binding) и их применение в MVC/MVVM паттернах.  
3. **Разработать и настроить контекстные меню** и горячие клавиши для ускорения пользовательского взаимодействия.  
4. **Создать и конфигурировать диалоговые окна** (файловые диалоги, предупреждения, пользовательские диалоги) с динамическим содержимым.  
5. **Развить аналитическое и исследовательское мышление** через оценку альтернативных технологий и проектирование собственных решений.

## Ключевые понятия

| Термин | Расширенное определение | Зачем это нужно |
|--------|------------------------|-----------------|
| Виджет-список | Компонент UI, отображающий коллекцию элементов (строк, объектов) с возможностью выбора, редактирования и сортировки. | Позволяет пользователю быстро просматривать и управлять данными. |
| Привязка данных (Data Binding) | Механизм, связывающий свойства UI с объектами модели, автоматически синхронизируя изменения. | Снижает количество boilerplate‑кода и повышает надёжность. |
| Контекстное меню | Появляется при правом клике и содержит контекстно‑релевантные действия. | Улучшает доступность функций без загромождения основного интерфейса. |
| Диалоговое окно | Появляющийся поверх основной сцены модальный или модельный элемент, требующий пользовательского ввода или подтверждения. | Предотвращает ошибки, уточняет намерения пользователя, обеспечивает согласованность. |
| MVC / MVVM | Паттерны проектирования, разделяющие логику представления, модели и контроллеров/вьюмоделей. | Делят ответственность, упрощают тестирование и поддержку. |
| Описательный язык разметки (XML/JSON) | Структурированный формат хранения конфигурации UI и данных. | Позволяет динамически генерировать интерфейсы и легко менять их. |
| События и колбэки | Механизм реагирования на пользовательские действия (клики, ввод). | Позволяют писать реактивный код, упрощают отладку. |
| Состояние (State) | Текущее представление данных и UI‑элементов. | Необходимо для корректного отображения и синхронизации. |
| Кроссплатформенность | Способность приложения работать на разных ОС без изменения кода. | Увеличивает аудиторию, снижает расходы на поддержку. |
| Тестируемость UI | Возможность писать автоматические тесты для интерфейса. | Повышает надёжность и ускоряет цикл разработки. |

## Примеры и иллюстрации

### 1. Список с привязкой в PyQt5

```python
import sys
from PyQt5.QtWidgets import QApplication, QMainWindow, QTableView
from PyQt5.QtCore import QAbstractTableModel, Qt

class SimpleTableModel(QAbstractTableModel):
    def __init__(self, data):
        super().__init__()
        self._data = data

    def rowCount(self, parent=None):
        return len(self._data)

    def columnCount(self, parent=None):
        return len(self._data[0])

    def data(self, index, role=Qt.DisplayRole):
        if role == Qt.DisplayRole:
            return self._data[index.row()][index.column()]
        return None

app = QApplication(sys.argv)
window = QMainWindow()
model = SimpleTableModel([["Alice", 30], ["Bob", 25]])
view = QTableView()
view.setModel(model)
window.setCentralWidget(view)
window.show()
sys.exit(app.exec_())
```

### 2. Кастомное диалоговое окно для выбора цвета

```python
from PyQt5.QtWidgets import QDialog, QVBoxLayout, QPushButton, QColorDialog

class ColorPickerDialog(QDialog):
    def __init__(self, parent=None):
        super().__init__(parent)
        self.setWindowTitle("Выберите цвет")
        layout = QVBoxLayout()
        self.btn = QPushButton("Показать диалог")
        self.btn.clicked.connect(self.open_color_dialog)
        layout.addWidget(self.btn)
        self.setLayout(layout)

    def open_color_dialog(self):
        color = QColorDialog.getColor()
        if color.isValid():
            self.parent().setStyleSheet(f"background-color: {color.name()};")
```
